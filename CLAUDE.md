# Правила проекта

## Удаление
- Ничего нельзя удалять без явного разрешения пользователя. Перед любым удалением (файлов, папок, данных, строк кода) — сначала объяснить на естественном языке, что именно и зачем планируется удалить, и дождаться подтверждения.

## Написание кода
- Перед написанием кода — сначала описать план на естественном языке, чтобы пользователь понял, что будет сделано, и подтвердил.
- Если код может что-то удалить (файлы, данные, записи и т.д.) — обязательно спросить у пользователя на естественном языке, выполнять это или нет. Не выполнять без подтверждения.

## Графики и сравнение
- Все эксперименты сравниваются с **Classic** режимом (эталон — воспроизведение Mathcad).
- Не более **2 графиков** на 1 изображении (например: Classic | Modern).

## Журнал экспериментов
- При каждой проверке идеи/гипотезы — записывать в `docs/experiment_log.md`:
  1. **Идея**: что проверяем
  2. **Описание**: обоснованность, математический аппарат, привязка к физике/эксперименту
  3. **Реализация**: что именно изменено в коде
  4. **Валидация**: результаты (графики, числа, сравнение)
  5. **Итог**: работает / не работает / частично
  6. **Следующие шаги**: что делать дальше
- Каждая итерация — отдельная запись с датой
- Файл: `docs/experiment_log.md`

## Документация проекта (Этап 0)

- `docs/mathcad2000_reference.md` — справочник по Mathcad 2000: формат MCD, синтаксис, встроенные функции
- `docs/algorithm_description.md` — детальное описание алгоритма обработки прописей: все этапы, формулы, параметры, коэффициенты растворимости, формулы SolutionMaker
- `docs/ershov_dissertation_summary.md` — конспект диссертации Ершова: теория интерферометрии, мёртвая зона, кинетика Аррениуса, влияние примесей
- `docs/research_improvements.md` — обзор литературы и предложения по улучшению: современные методы, BCF-модель, автоматизация, обновление эталонов
- `docs/research_signal_cleaning.md` — исследование очистки сигнала: устранение осцилляций arcsin d-step, CWT Ridge/EMD/PLL методы извлечения фазы, правильная предобработка (highpass вместо bandpass), robust-фиттинг, обработка автокорреляции
- `docs/mathcad_full_extraction.md` — **ПОЛНАЯ экстракция алгоритма из Mathcad** (32 скриншота `mathcad_print/`): все формулы, переменные, порядок выполнения, коэффициенты эталонных кривых (a/a1/b/b1), матрицы D/DFe, zs-формула скорости, arcsin d-step фаза, grid search фит, Sig035, s2, формула tn с коррекцией
- `docs/audit_vs_mathcad.md` — **Аудит Python vs Mathcad** (полный пересмотр 2026-03-19): все 30 скриншотов прочитаны, Classic ~99% совпадает с Mathcad; 5 критических/средних багов ИСПРАВЛЕНЫ; осталось мелкое отличие x0 в zs-формуле (~0.2% влияния)
- `docs/research_dead_zone_modern.md` — **Исследование dead zone в Modern pipeline**: почему Hilbert даёт ненулевую R при dT=0, сравнение методов (CWT Ridge/PLL/EMD), варианты решения (envelope masking, Td cutoff, CWT Ridge), анализ применимости модели Кабреры-Вермильи (CV) к KDP/DKDP + Fe³⁺ (все грани, 10–70°C, ±кислота), рекомендации
- `docs/research_dual_channel.md` — **Двухканальная обработка**: dual-channel Hilbert, CMR, квадратурная демодуляция, PLL, CWT Ridge, когерентность, биморфизм KDP — что можно и нельзя извлечь из LED1+LED2
- `docs/research_amplitude_normalization.md` — **Нормализация амплитуды**: peak-trough envelope, связь амплитуды и фазы, почему Hilbert шумит в dead zone, Вариант D (peak-trough + Hilbert)
- `docs/research_tn_key_methods.md` — **Научный обзор ключевых методов**: определение критических параметров фазовых переходов из косвенных измерений — 5 глав (распространение ошибки, dead zone / CV-модель, выбор кинетической модели BCF/power law/B+S, Simulation-Based Inference, символьная регрессия PySR/SINDy/KAN) + интеграция + краткие обзоры NLME/bootstrap/GP/PINN/conformal prediction/changepoint detection/Neural ODE + дорожная карта
- `docs/experiment_log.md` — **Журнал экспериментов**: итерации по улучшению Modern pipeline, идеи, валидация, результаты
