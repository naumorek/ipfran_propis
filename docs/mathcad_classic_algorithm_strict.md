## 1. Физическая постановка задачи

### 1.1. Что такое пропись

**Пропись** (она же «прогон», «цикл измерений») — это запись показаний интерферометрического
стенда во время управляемого изменения температуры раствора KDP или DKDP. Стенд измеряет
скорость роста (или растворения) затравочного кристалла, находящегося в кювете с раствором.

Типичный сценарий прописи:
1. Раствор охлаждается ниже температуры насыщения — кристалл **растёт**
2. Раствор нагревается выше температуры насыщения — кристалл **растворяется**

В процессе регистрируется интерференционный сигнал: два светодиода (LED) с разными длинами
волн освещают грань кристалла, и при изменении толщины слоя возникают интерференционные
полосы — синусоидальные осцилляции интенсивности.

### 1.2. Что нужно определить

Из одной прописи алгоритм извлекает:

| Параметр | Обозначение | Единицы | Физический смысл |
|----------|-------------|---------|------------------|
| Температура равновесия | `te` | °C | Температура при которой начинается растворение (определяется по границе `isat` в данных) |
| Температура насыщения | `tn` | °C | Истинная температура насыщения раствора (свойство раствора, одинаковая для всех прописей одного дня) |
| Мёртвая зона (в °C) | `Td` | °C | Область переохлаждения, в которой кристалл не растёт: `Td = te - tm` |
| Мёртвая зона (в %) | `s1` | % (σ) | То же самое, но выражено через относительное пересыщение |
| Пересыщение при Td | `Sigm` | % | Относительное пересыщение, соответствующее мёртвой зоне |
| Показатель кинетики | `s2` | безразм. | Характеризует форму кинетической кривой (см. раздел 8) |
| Порог Sig035 | `Sig035` | % (σ) | Пересыщение, при котором скорость роста достигает 0.35 мм/день |
| Кинетическая кривая | `R(σ)` | мм/день | Зависимость скорости роста от пересыщения |

### 1.3. Интерференционный принцип измерения

Кристалл в растворе освещается светодиодом с длиной волны λ. Отражённый от поверхности
раствора и от грани кристалла лучи интерферируют. При росте (или растворении) кристалла
его грань смещается, и интенсивность света осциллирует синусоидально.

Каждый полный период синусоиды (от максимума до следующего максимума) соответствует
смещению грани кристалла на величину:

```
Δh = λ / (2n)
```

где `n` — показатель преломления раствора при данной длине волны.

Частота осцилляций пропорциональна скорости роста: чем быстрее растёт кристалл,
тем чаще «мигает» сигнал. При растворении — то же самое, но фаза меняется
в другую сторону. При нулевой скорости — сигнал постоянный (нет осцилляций).

### 1.4. Грани кристалла

KDP/DKDP кристаллы имеют форму тетрагональной бипирамиды. Измерения проводятся
для двух типов граней:
- **Призма** (`Face = 0`) — боковые грани, обычно растут медленнее
- **Пирамида** (`Face = 1`) — торцевые грани, обычно растут быстрее

Тип грани влияет на оптические коэффициенты (`gran1`, `gran2`).

## 2. Входные данные

### 2.1. Файл PRN

Текстовый файл, ~65000 строк, 7 столбцов (в Mathcad индексируются 0–6).
Частота дискретизации: 1 отсчёт/секунду (~18 часов записи).

| Столбец | Mathcad | Обозначение | Физический смысл |
|---------|---------|-------------|------------------|
| 0 | `a[i,0]` | — | Порядковый номер отсчёта (не используется в вычислениях) |
| 1 | `a[i,1]` | `S_raw` | Сигнал светодиода 1 (основной интерференционный сигнал), Вольты |
| 2 | `a[i,2]` | — | Сигнал светодиода 2 (вспомогательный), Вольты |
| 3 | `a[i,3]` | — | Флаг/статус (всегда 0.00, не используется) |
| 4 | `a[i,4]` | `T_raw` | Температура (условные единицы: ~273 + T°C) |
| 5 | `a[i,5]` | `T_°C` | Температура раствора, °C |
| 6 | `a[i,6]` | `time` | Время (формат HH:MM:SS, не используется напрямую) |

**Температура в Mathcad:** всюду используется столбец 4, пересчитанный как:
```
T(i) = a[i, 4] - 273.15    [°C]
```

### 2.2. Ручные параметры оператора

Оператор задаёт параметры визуально, глядя на график сигнала. Это ключевая особенность
классического алгоритма — он **полуавтоматический**, требует эксперта.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `n1`, `n2` | int | Начало и конец интересующего участка записи (индексы отсчётов). Обычно один «цикл» нагрев-охлаждение |
| `im1` | int | Грубая граница начала мёртвой зоны (индекс). Определяется визуально как момент, когда осцилляции начинают затухать |
| `isat1` | int | Грубая граница начала растворения (индекс). Момент, когда осцилляции вновь появляются (но уже в зоне растворения) |
| `im` | int | Точная граница мёртвой зоны (индекс). Уточнённая позиция начала затухания |
| `isat` | int | Точная граница начала растворения (индекс). Уточнённая позиция |
| `Salt` | 1 или 2 | Тип соли: 1 = KDP, 2 = DKDP |
| `Acid` | 0 или 1 | Тип раствора: 0 = нейтральный, 1 = кислый |
| `Face` | 0 или 1 | Тип грани: 0 = призма, 1 = пирамида |
| `l` | float | Коэффициент сглаживания (обычно 0.01). Используется в LOESS для огибающей |
| `dtau` | float | Промежуток времени между отсчётами в минутах (обычно 0.055 мин ≈ 3.3 сек). **Не равен 1 сек!** Это время между группами отсчётов |
| `d` | float | Шаг разбиения сигнала на сегменты (обычно 3.3). Используется при расчёте угловых координат |
| `ww` | int | Весовой коэффициент (обычно 1) |
| `k1` | float | Коэффициент окрестности для регрессии экстремумов (обычно 0.15) |
| `L0` | float | Базовое число интерференционных полос (обычно 1500) |
| `span1` | float | Ширина окна для LOESS-сглаживания R(σ) (обычно 0.15–0.2) |
| `vq` | 0 или 1 | Направление обхода: 0 = прямой (нагрев), 1 = обратный |

### 2.3. Физический смысл границ

```
     n1                    im1    im  isat isat1                   n2
      |                      |    |    |    |                       |
      |    РОСТ КРИСТАЛЛА    | МЗ |    |    РАСТВОРЕНИЕ            |
      |  (быстрые осцилляции)|    |    |  (осцилляции снова)       |
      |______________________|____|____|____|_______________________|

      T: низкая ──────────────────────────────────────────── высокая
      R: высокая ─────────── затухает → 0 → появляется ──── высокая
```

- **Рост** (от `n1` до `im1`): температура ниже `tn`, кристалл растёт, видны осцилляции
- **Мёртвая зона** (от `im1`/`im` до `isat`/`isat1`): температура поднялась, но ещё ниже `te`;
  рост прекратился, осцилляций нет (мёртвая зона — из-за барьера нуклеации)
- **Растворение** (от `isat`/`isat1` до `n2`): температура выше `te`, кристалл растворяется,
  осцилляции появляются снова

`im1`/`isat1` — грубые границы (для расчёта средних), `im`/`isat` — точные (для фиттинга).

## 3. Предварительная обработка данных

### 3.1. Вырезка рабочего участка

```
a := submatrix(a, n1, n2, 0, 5)
m := rows(a)
```

Из полного файла (~65000 строк) берётся только участок от `n1` до `n2`.
После этого все индексы считаются от 0 до `m-1` внутри вырезки.

### 3.2. Определение направления обхода

```
nn := -1    если vq > 0   (обратный ход)
nn :=  1    иначе          (прямой ход)
```

`nn` определяет знак скорости роста: при прямом ходе (нагрев) рост положителен,
при обратном — отрицателен. Далее скорость домножается на `nn`.

### 3.3. Вычисление температуры

```
te := a[isat, 4] - 273.15
```

Температура `te` берётся из строки `isat` (точная граница начала растворения).
Это **температура при которой начинается растворение** в данной прописи.

```
amin := min(a[*, 4]) - 273.15    — минимальная температура
amax := max(a[*, 4]) - 273.15    — максимальная температура
```

### 3.4. Медианное сглаживание сигнала

```
S1 := medsmooth(a[*, 1], 5)
```

Сигнал LED1 (столбец 1) сглаживается медианным фильтром с окном 5 отсчётов.
Это убирает импульсные выбросы (спайки), сохраняя форму синусоиды.

`medsmooth(v, n)` — встроенная функция Mathcad: для каждого отсчёта берёт медиану
из `n` соседних значений.

### 3.5. Вычисление средних уровней сигнала

```
y0  := (1 / im1) · Σ_{i=0}^{im1-1} S1[i]         — среднее в зоне роста
y0s := (1 / (m - isat1)) · Σ_{i=isat1}^{m-1} S1[i] — среднее в зоне растворения
```

Средние `y0` и `y0s` — это «нулевые линии» (baseline), относительно которых
определяются экстремумы синусоиды. Экстремумы — это отклонения от этой средней.

**Почему два средних:** амплитуда и постоянная составляющая сигнала могут отличаться
в зоне роста и в зоне растворения (из-за изменения условий — температура,
оптический путь, дрейф электроники).

## 4. Детекция экстремумов синусоиды (зона растворения)

Этот шаг — **ключевой** для всего алгоритма. Из осцилляций сигнала извлекаются
позиции и значения каждого максимума и минимума. По ним затем строится фазовая
функция и вычисляется скорость роста.

### 4.1. Зачем нужны экстремумы

Интерференционный сигнал — это приблизительная синусоида с переменным периодом
(период зависит от скорости роста) и переменной амплитудой (затухание с удалением
от точки насыщения). Каждая пара соседних экстремумов (max-min или min-max)
соответствует прохождению фазы на π (полупериод), т.е. изменению толщины кристалла
на `λ/(4n)`.

### 4.2. Алгоритм грубой детекции (zero-crossings)

Работает **только в зоне растворения** (от `isat1` до конца записи).

**Вход:** сглаженный сигнал `S1`, среднее `y0s`, начальный индекс `isat1`.

**Алгоритм (псевдокод):**

```
function find_extrema(S1, y0s, isat1, m):
    ma := -10           # позиция текущего максимума отклонения
    mb := -10           # значение текущего максимального отклонения

    # Определить начальный знак (выше или ниже средней)
    if S1[isat] - y0s > 0:
        k := +1         # сейчас выше средней → ищем пересечение сверху вниз
    else:
        k := -1         # сейчас ниже средней → ищем пересечение снизу вверх

    j := 0              # счётчик найденных экстремумов
    результат := []     # массив (позиция, значение)

    for i := isat1, isat1+2, isat1+4, ... до m-1:     # ШАГ 2!
        отклонение := |S1[i] - y0s|

        # Обновить максимум отклонения в текущей полуволне
        if отклонение > mb:
            ma := i
            mb := отклонение

        # Проверить смену знака (пересечение средней линии)
        if (S1[i] - y0s) · k < 0:
            # Зафиксировать экстремум: позиция ma, значение S1[ma]
            результат[j] := (ma, S1[ma])
            j := j + 1
            mb := -1       # сбросить отслеживание максимума
            k := -k        # переключить знак (теперь ищем в другую сторону)

    # Обработка последнего экстремума (если он далеко от конца)
    if m - 1 - ma > 4:
        результат[j] := (ma, mb)

    return результат
```

**Суть:** Идём по сигналу с шагом 2. В каждой полуволне (между пересечениями средней
линии `y0s`) запоминаем позицию максимального отклонения от средней. При смене знака
`(S1[i] - y0s)` — фиксируем экстремум и начинаем искать следующий.

**Шаг 2** (через один отсчёт) — для ускорения и фильтрации высокочастотного шума.

**Результат:** массив `extrs` размером `h × 2`:
- `extrs[j, 0]` — индекс (позиция) j-го экстремума в массиве данных
- `extrs[j, 1]` — значение сигнала S1 в этой позиции

Типичное число экстремумов: `h ≈ 26` для стандартной прописи.

### 4.3. Фильтрация крайних экстремумов

После грубой детекции убираются ненадёжные экстремумы на краях:

```
kk := rows(extrs) - 1     # индекс последнего

# 1. Первый слишком близко к isat1?
if extrs[0, 0] - isat1 ≤ 3:
    убрать первый

# 2. Последний слишком близко к концу?
if m - extrs[kk, 0] - 1 ≤ 4:
    убрать последний

# 3. Последний экстремум имеет отрицательное значение?
if extrs[kk, 1] ≤ 0:
    убрать последний

# 4. Итого: ks = rows(extrs) - 1 (число интервалов между экстремумами)
```

### 4.4. Уточнение позиций экстремумов квадратичной регрессией

Грубая детекция даёт позиции с точностью ±2 отсчёта (из-за шага 2).
Для повышения точности вокруг каждого грубого экстремума берётся окрестность
и аппроксимируется параболой (полиномом 2-й степени).

**Для каждого экстремума j (j = 0..ks):**

```
# Определить размер окрестности
if j < ks:
    n := floor(|extrs[j+1, 0] - extrs[j, 0]| · k1)     # k1 = 0.15
else:
    n := floor(|extrs[j-1, 0] - extrs[j, 0]| · k1)

# Определить границы окрестности
if j < 1:
    u := extrs[j, 0] - floor(n/2)    # для первого — симметричная окрестность
else:
    u := extrs[j, 0] - n              # для остальных — несимметричная (больше назад)

v := min(extrs[j, 0] + n, m - 1)

# Собрать данные окрестности
for i := u..v:
    x[i-u] := i - u        # локальная координата (0, 1, 2, ...)
    y[i-u] := S1[i]        # значение сигнала

# Квадратичная регрессия: y = z₃ + z₄·x + z₅·x²
z := regress(x, y, 2)      # Mathcad regress с degree=2

# Вершина параболы (точный экстремум)
es[j, 0] := -z[4] / (2·z[5]) + u     # абсолютная позиция (дробная!)
es[j, 1] := z[3] + z[4]·(-z[4]/(2·z[5])) + z[5]·(-z[4]/(2·z[5]))²  # значение
```

**Зачем:** Парабола точно проходит через вершину синусоиды, даже если она не попала
ни на один дискретный отсчёт. Это даёт субдискретную точность положения экстремума.

**Примечание о regress:** Функция `regress(x, y, degree)` в Mathcad возвращает вектор
коэффициентов. Для degree=2: `z[3] = a₀`, `z[4] = a₁`, `z[5] = a₂`, где `y = a₀ + a₁x + a₂x²`.

**Результат:** массив `es` размером `(ks+1) × 2`:
- `es[j, 0]` — точная (дробная) позиция j-го экстремума
- `es[j, 1]` — точное значение сигнала в экстремуме

## 5. Построение непрерывной фазовой функции

### 5.1. Зачем нужна фаза

Простой подсчёт числа экстремумов даёт грубую оценку скорости. Но для точного
построения кинетической кривой нужна **непрерывная фазовая функция** Φ(t),
значение которой растёт на π при каждом проходе через экстремум.

Скорость роста R = dΦ/dt · (λ / (2πn)) · пересчётные_коэффициенты.

### 5.2. Разбиение сигнала на сегменты и вычисление фазы

Алгоритм разбивает область данных на сегменты длиной `d` (обычно 3.3 отсчёта)
и для каждого сегмента вычисляет фазу через арксинус, используя значения
в ближайших экстремумах как масштаб.

**Шаг 1: Подготовка интерполяционных функций**

```
tau := массив индексов 0, 1, 2, ..., m-1
I_values := S1[0], S1[1], ..., S1[m-1]    # интерполяция сигнала

# Функция I(x) — интерполяция сигнала в дробной точке x:
I(x) := linterp(tau, I_values, x)

# Время в минутах (используя столбец 4 как метку времени):
t_values := a[0,4], a[1,4], ..., a[m-1,4]
t(x) := linterp(tau, t_values, x)

# Температура (через интерполяцию):
Tau(x) := linterp(tau, al, x)
# где al[v] = a[v, 0] (не путать с температурой — это столбец 0, номер отсчёта)
```

**Шаг 2: Определение числа полных полуволн**

```
g := floor(isat1 / d)     # число сегментов до начала растворения
ss := ks - 1               # число полных полуволн (между соседними экстремумами)
s := h - 1                 # общее число интервалов
```

**Шаг 3: Вычисление угловых координат (ys)**

Для каждого r-го сегмента длиной `d`, между экстремумами `es[i]` и `es[i-1]`:

```
ys := []
j := 0

# Часть 1: зона ДО первого экстремума (от im1 до es[0])
for r := 0..ks-1:
    pos := im1 + (es[0,0] - im1) / ks · r
    pos := floor(pos)

    # Определить, между какими экстремумами мы находимся
    # (здесь используется поиск ближайшего es)

    if r > 0:
        # Фаза через арксинус:
        ys[j, 1] := π/2·(s+1-i) - arcsin((S1[pos] - es[i,1]) / (es[i-1,1] - es[i,1]))
    else:
        ys[j, 1] := π/2·(s+1-i)    # точно в экстремуме — фаза кратна π/2

    ys[j, 0] := pos
    j := j + 1

# Часть 2: зона между экстремумами (основная)
for i := 0..ss:
    for r := 0..es[i, 2]-1:
        pos := es[i+1,0] - (es[i+1,0] - es[i,0]) / es[i,2] · r
        pos := floor(pos)

        if r > 0:
            # Внутри полуволны: значение между es[i,1] и es[i-1,1]
            # arcsin нормализует сигнал к [-1, 1]
            ys[j, 1] := π/2·(s+1-i) - arcsin((S1[pos] - es[i,1]) / (es[i-1,1] - es[i,1]))
        else:
            ys[j, 1] := π/2·(s+1-i)

        ys[j, 0] := pos
        j := j + 1

# Последняя точка
ys[j, 0] := es[ss+1, 0]
ys[j, 1] := π/2·(s - ss)
```

**Ключевая формула фазы:**

```
Φ(pos) = π/2 · (s + 1 - i) - arcsin( (S1[pos] - es[i, 1]) / (es[i-1, 1] - es[i, 1]) )
```

Здесь:
- `s + 1 - i` — номер полуволны от конца (убывает к началу записи)
- Аргумент arcsin — нормализованное отклонение сигнала от ближайшего экстремума,
  масштабированное на разницу соседних экстремумов (max-min)
- Это преобразование «распрямляет» синусоиду в монотонную функцию

**Физический смысл:** Фаза Φ монотонно растёт от 0 (начало роста) до `π/2 · (ks+1)`
(конец растворения). Скорость нарастания фазы пропорциональна скорости роста кристалла.

## 6. Вычисление скорости роста и температуры

### 6.1. Матрица z — основной результат

Для каждого интервала между соседними точками `ys` вычисляется:

```
for i := 0..rows(ys)-2:
    # Средняя позиция
    z[i, 0] := (ys[i, 0] + ys[i+1, 0]) / 2

    # Средняя температура в интервале
    y1 := ceil(ys[i, 0])
    y2 := floor(ys[i+1, 0])
    z[i, 2] := (1 / (y2 - y1 + 1)) · Σ_{r=y1}^{y2} (a[r, 4] - 273.15)

    # Скорость роста
    z[i, 1] := (Φ[i+1] - Φ[i]) / (Tau(ys[i+1, 0]) - Tau(ys[i, 0])) · 30 · nn
```

Здесь `Φ[i]` — это `ys[i, 1]` (фаза в i-й точке), а `Tau` — время.

### 6.2. Более точная формула скорости (из файла _040326_6_2)

В полном алгоритме формула скорости учитывает показатель преломления:

```
x0 := co₀ - co₁ · (a[m-1, 4] - 273.15)
x1 := co₀ - co₁ · (a[y1, 4] - 273.15)
x2 := co₀ - co₁ · (a[y2, 4] - 273.15)

z[i, 1] := [ L1·x0·(1/x2 - 1/x1) + (1/π)·(ys[i+1,1]/x2 - ys[i,1]/x1) ]
            · 30 / ((ys[i+1,0] - ys[i,0]) · ww)
```

Где:
- `L1 = -Q / (π · co₁)` — нормировочная константа, связанная с длиной волны
- `co₀, co₁` — оптические коэффициенты (зависят от Salt, Face)
- `x0, x1, x2` — показатели преломления при разных температурах
- `30` — пересчёт в мм/день (из единиц фаза/мин)

**Упрощённая формула** (без учёта преломления, когда x1 ≈ x2 ≈ x0):

```
R ≈ (ΔΦ / Δt) · λ/(2π·n) · коэфф_пересчёта
```

### 6.3. Вычисление пересыщения

Для каждой точки `z` вычисляется **относительное пересыщение** σ%:

```
z[i, 3] := 100 · ln(C_n / C(T_i))
```

Где:
- `C_n = co₂ + co₃·tn + co₄·tn²` — растворимость при температуре насыщения
- `C(T_i) = co₂ + co₃·T_i + co₄·T_i²` — растворимость при текущей температуре `T_i = z[i,2]`
- `ln` — натуральный логарифм

**Физический смысл:** σ > 0 означает пересыщение (раствор «хочет» кристаллизоваться),
σ < 0 — недосыщение (кристалл растворяется).

### 6.4. Графики, формируемые на этом этапе

**График 1: Сырой сигнал (скр. 1, 3-4)**
- Ось X: номер отсчёта (0 ... m)
- Ось Y: напряжение S1 (Вольты)
- Красные точки: позиции экстремумов (из `extrs`)
- Зелёные линии: положения `im1` и `isat1`
- Физический смысл: показывает интерференционные полосы. Густые осцилляции = быстрый рост.
  Область без осцилляций = мёртвая зона.

**График 2: Сигнал в зоне мёртвой зоны (скр. 5)**
- Увеличенный вид зоны im1..isat1
- Показывает, что сигнал «вертикален» (нет осцилляций) — кристалл не растёт

**График 3: Фазовая функция (скр. 5, нижний левый)**
- Ось X: номер отсчёта
- Ось Y: значение фазы Φ
- Монотонно убывающая кривая (от максимального роста к нулю)

## 7. Мёртвая зона и температура насыщения

### 7.1. Температура при начале растворения (te)

```
te := a[isat, 4] - 273.15     [°C]
```

Берётся напрямую из данных в точке `isat`.

### 7.2. Температура начала мёртвой зоны (tm)

```
tm := a[im, 4] - 273.15 - 0.01·(te - 25)     [°C]
```

**Поправка** `-0.01·(te - 25)` — эмпирическая коррекция, учитывающая систематическую
погрешность определения начала мёртвой зоны при разных температурах.

### 7.3. Мёртвая зона в градусах (Td)

```
Td := te - tm     [°C]
```

Это разница между температурой начала растворения и температурой начала мёртвой зоны.
Типичные значения: 0.1–0.5 °C.

### 7.4. Температура насыщения (tn)

`tn` — **свойство раствора**, одинаковое для всех прописей одного дня.
Определяется оператором отдельно (обычно из серии прописей).

В Mathcad-файле `tn` задаётся как `tn := ■` (ручной ввод) или вычисляется
по линейной экстраполяции (см. раздел 8).

### 7.5. Концентрации и пересыщение мёртвой зоны

```
C_n := co₂ + co₃·te + co₄·te²     — концентрация при te (≈ при насыщении)
C_m := co₂ + co₃·tm + co₄·tm²     — концентрация при tm

Sigm := 100 · ln(C_n / C_m)        — пересыщение в мёртвой зоне, %
```

`Sigm` ≈ 0.3% — типичное значение для KDP.

## 8. Аппроксимация кинетической кривой R(σ)

Это центральный этап — из точечных значений `(σ_i, R_i)` строится гладкая
кинетическая кривая и определяются её параметры.

### 8.1. Подготовка данных

```
u := csort(z, 3)           # сортировка матрицы z по столбцу 3 (σ%)

j := 0..rows(u)-1
VX[j] := u[j, 3]           # σ%, отсортированное по возрастанию
VY[j] := u[j, 1]           # R (мм/день), соответствующее данному σ

span1 := 0.2
S3 := loess(VX, VY, span1)         # LOESS-сглаживание
F1(x) := interp(S3, VX, VY, x)     # интерполяция результата LOESS
```

**LOESS** (LOcally Estimated Scatterplot Smoothing) — метод локально-взвешенной
полиномиальной регрессии. `span = 0.2` означает, что для сглаживания каждой точки
используется 20% ближайших данных.

**F1(σ)** — гладкая функция, аппроксимирующая зависимость R(σ).

### 8.2. Отсечение нулевых скоростей

Точки с R ≤ 0.01 удаляются перед фиттингом (они относятся к мёртвой зоне):

```
VX1, VY1 := VX, VY  с удалённым начальным участком где VY ≤ 0.01
n := rows(VX1) - 1      # число оставшихся точек (типично ~261)
```

### 8.3. Фиттинг степенным законом (определение s₀ и s₁)

Кинетическая модель:

```
G(σ) = s₀ · (σ - s₁)^w      если σ > s₁
G(σ) = 0                      если σ ≤ s₁
```

Где:
- `σ` = σ% (пересыщение в процентах, т.е. VX)
- `s₁` = мёртвая зона в %, точка отсечения (может быть отрицательной!)
- `s₀` = кинетический коэффициент
- `w` = показатель степени (обычно w = 1, линейная кинетика)

**Алгоритм фиттинга (полный перебор):**

```
w := 1
min_f := 100000

for i := 0..n-1:
    for x := VX1[i], VX1[i]+0.01, VX1[i]+0.02, ..., VX1[i+1]:

        # Для данного порога x, вычислить оптимальный s₀:
        q1 := Σ_{k=i}^{n} (VX1[k] - x)^w · VY1[k]
        q2 := Σ_{k=i}^{n} (VX1[k] - x)^(2w)
        q := q1 / q2          # оптимальный s₀ для данного x

        # Вычислить остаток (невязку):
        f := Σ_{k=0}^{i} (VY1[k])²                                    # точки ниже порога: модель = 0
           + Σ_{k=i}^{n} [q · (VX1[k] - x)^w - VY1[k]]²             # точки выше порога: отклонение от модели

        # Обновить лучшее решение
        if f < min_f:
            s₀ := q
            s₁ := x
            min_f := f

# Результат:  s = (s₀, s₁)
# Пример: s = (-0.0129, -0.4305)
```

**Как это работает:**
1. Перебираем все возможные пороги `x` (мёртвая зона) от VX1[0] до VX1[n] с шагом 0.01
2. Для каждого порога аналитически находим оптимальный `s₀` (линейная МНК по одному параметру)
3. Вычисляем суммарную невязку
4. Выбираем порог с минимальной невязкой

**Формула для модельной кривой (для графика):**

```
h := 0..floor(VX[hh] · 100)        # hh = rows(VX) - 1
G[h] := s₀ · (h·0.01 - s₁)^w      если h·0.01 > s₁
G[h] := 0                           иначе
```

Здесь `h·0.01` переводит индекс в значение σ%.

### 8.4. Определение s₂ (показатель формы кинетической кривой)

**s₂** характеризует отклонение реальной кинетики от модели. Определяется
через линейную регрессию в координатах `√R vs σ%`.

**Алгоритм:**

```
# Шаг 1: Собрать точки из z (НЕ из отсортированного u!)
Q := []
j := 0
for i := 0..rows(z)-1:
    if z[i, 1] < 0:     # отрицательная скорость — стоп
        break
    if z[i, 1] < 0.3:   # только точки с R < 0.3 мм/день
        Q[j, 0] := z[i, 3]          # σ%
        Q[j, 1] := √(z[i, 1])       # √R
        j := j + 1

# Шаг 2: Линейная регрессия √R = Q1 + Q2·σ%
Q1 := intercept(Q[*, 0], Q[*, 1])    # свободный член
Q2 := slope(Q[*, 0], Q[*, 1])        # наклон

# Шаг 3: s₂ — точка пересечения прямой с осью σ
s2 := -Q1 / Q2
```

**Физический смысл s₂:**
- Это значение σ%, при котором экстраполированная прямая `√R = Q1 + Q2·σ%`
  пересекает ось абсцисс (`√R = 0`, т.е. `R = 0`)
- По сути, это **ещё одна оценка мёртвой зоны**, полученная другим методом (в координатах √R)
- Отрицательное s₂ означает, что прямая пересекает ось при σ < 0 (слабая мёртвая зона)
- Типичное значение: s₂ ≈ -0.04

**Почему √R, а не R:** В теории (модель BCF/Кабрера-Вермилье), для спиральной
кинетики R ~ σ² при малых σ, поэтому √R ~ σ — линейная зависимость. Линейная
регрессия √R vs σ даёт устойчивую оценку мёртвой зоны.

**Фильтр R < 0.3:** Используются только точки с малой скоростью (близкие к мёртвой зоне),
где линейная аппроксимация √R ≈ Q1 + Q2·σ работает лучше всего.

**Фильтр R < 0:** Остановка при отрицательной скорости — признак перехода
в зону растворения, эти точки не нужны.

### 8.5. Определение Sig035

**Sig035** — значение пересыщения σ%, при котором LOESS-сглаженная скорость роста
впервые превышает 0.35 мм/день.

```
Sig035 := ?
for i := 0..rows(u)-1:
    Sig035 := VX[i]
    if F1(VX[i]) > 0.35:
        break
```

Здесь `F1` — LOESS-интерполяция из шага 8.1. Проход по отсортированным значениям
σ% от наименьшего. Первое σ%, при котором сглаженная скорость > 0.35, становится Sig035.

**Физический смысл:** Sig035 показывает, какое пересыщение нужно, чтобы кристалл
рос со скоростью 0.35 мм/день. Чем меньше Sig035 — тем лучше раствор
(кристалл начинает расти при меньшем переохлаждении).

### 8.6. Определение te1 (линейная экстраполяция)

```
# Линейная регрессия R(σ) в зоне малых скоростей:
# Берём точки VX1, VY1 (после отсечения нулей)

as := slope(VX1, VY1)
bs := intercept(VX1, VY1)
te1 := -bs / as
```

Это экстраполяция на R = 0 линейного участка кинетической кривой.
Даёт оценку мёртвой зоны ещё одним методом. Используется как вспомогательный параметр.

### 8.7. Графики, формируемые на этом этапе

**График 4: R(ΔT) — кинетическая кривая в градусах (скр. 8)**
- Ось X: ΔT = te - T, переохлаждение в °C (от 0 вправо)
- Ось Y: R, скорость роста в мм/день
- Чёрные точки: экспериментальные данные z[i]
- Чёрная кривая: аппроксимация G(σ) = s₀·(σ - s₁)^w
- Зелёная кривая: эталон Cfe=0 (чистый раствор — наилучшая кинетика)
- Красная кривая: эталон Cfe=16ppm (загрязнённый раствор — наихудшая)
- **Интерпретация:** Если чёрная кривая между зелёной и красной — раствор нормального качества.
  Левее зелёной — отличный. Правее красной — непригодный для роста.

**График 5: R(σ%) — кинетическая кривая в процентах пересыщения (скр. 9-10)**
- Ось X: σ%, пересыщение
- Ось Y: R, мм/день (или мкм/мин в нижнем графике)
- Те же кривые, но в другом масштабе

**График 6: √R vs σ% (неявно, скр. 7)**
- Не рисуется явно, но Q-данные используются для определения s₂
- Прямая √R = Q1 + Q2·σ% — визуализация линейного фита

## 10. Эталонные кривые

### 10.1. Назначение

Экспериментальная кинетическая кривая сравнивается с двумя эталонами:
- **Cfe = 0** (чистый раствор) — кривая для раствора без примеси железа
- **Cfe = 16 ppm** (загрязнённый) — кривая для раствора с 16 ppm Fe³⁺

Эталоны получены эмпирически из серии измерений при разных температурах.

### 10.2. Матрицы D и DFe

Эталоны хранятся в виде матриц `D` (Cfe=0) и `DFe` (Cfe=16ppm).

Первая строка = значения σ%: `0, 0, 0.012, 0.016, 0.021, 0.029, 0.038, 0.051, 0.068, 0.09, 0.12, 0.16, 0.214, 0.285, 0.38, 0.506, 0.675, 0.9, 1.2, 1.6, ...`

Столбец 0 строк 1–4 = температуры: `19.88, 29.82, 40.48, 50.64` (для D).

Элемент `D[k, j]` = скорость роста R при температуре `D[k, 0]` и пересыщении `D[0, j]`.

### 10.3. Интерполяция эталона по температуре

Для текущей температуры `te` эталонные кривые интерполируются линейно:

```
Z := interpolate_reference(D, te)
# Для каждого столбца j:
#   Найти строки k-1, k такие что D[k-1, 0] < te ≤ D[k, 0]
#   x := (te - D[k-1, 0]) / (D[k, 0] - D[k-1, 0])
#   Z[j] := D[k-1, j] + (D[k, j] - D[k-1, j]) · x
```

Аналогично для `ZFe` (из `DFe`).

### 10.4. Аппроксимация Sig(t) — полиномиальные кривые

4 набора коэффициентов для аппроксимации эталонных кривых полиномами от температуры:

```
a  — KDP, Cac=9.8%, Cfe=4.5ppm     (8 строк × 4 столбца)
a1 — KDP, Cac=9.8%, Cfe=20.5ppm    (8 строк × 4 столбца)
b  — KDP, Cac=0, Cfe=0             (8 строк × 4 столбца)
b1 — KDP, Cac=0, Cfe=16ppm         (8 строк × 4 столбца)
```

Формула: для j-й точки кривой (j = 0..7):
```
Si[j, 0] := a[j,0] + a[j,1]·te + a[j,2]·te²     — σ% при данной скорости роста
Si[j, 1] := a[j, 3]                                — значение R (мм/день): 0, 0.025, 0.05, 0.1, 0.2, 0.4, 0.8, 1.6
```

Выбор набора коэффициентов:
```
if Acid < 1: используем b (без кислоты)
else:        используем a (с кислотой)
```

## 11. Коэффициенты растворимости

### 11.1. Кривая растворимости C(T)

Растворимость KDP/DKDP зависит от температуры:

```
C(T) = co₂ + co₃·T + co₄·T²
```

Где T — температура в °C, C — массовая доля (безразмерная).

### 11.2. Оптические коэффициенты co₀, co₁

Показатель преломления раствора:
```
n(T) = co₀ - co₁·T
```

Зависят от типа соли и грани:
```
gran1 := if Face > 0: 0.47854 else: 1.0
gran2 := if Face > 0: 0.9969  else: 1.0

if Salt = 1 (KDP):
    co₀ := 0.06446 · gran1
    co₁ := 1.92×10⁻⁵ · gran2
else (DKDP):
    co₀ := 0.061413 · gran1
    co₁ := 1.02393×10⁻⁵ · gran2
```

### 11.3. Коэффициенты растворимости co₂, co₃, co₄

| Salt | Acid | co₂ | co₃ | co₄ |
|------|------|-----|-----|-----|
| KDP (1) | нейтр (0) | 0.123 | 0.002719 | 1.1087×10⁻⁵ |
| KDP (1) | кислый (1) | 0.15 | 0.0032487 | 0 |
| DKDP (2) | нейтр (0) | 0.166 | 0.0037 | 0 |
| DKDP (2) | кислый (1) | 0.2061 | 0.003171 | 0 |

## 14. Словарь терминов

| Термин | Описание |
|--------|----------|
| **KDP** | Калий дигидрофосфат, KH₂PO₄. Нелинейный оптический кристалл |
| **DKDP** | Дейтерированный KDP, KD₂PO₄. Аналог KDP с дейтерием |
| **Пропись** | Один цикл записи данных со стенда (нагрев или охлаждение) |
| **Стенд** | Интерферометрическая установка для измерения скорости роста in situ |
| **Мёртвая зона** | Область переохлаждения (Td, °C) или пересыщения (s₁, Sigm, %), в которой скорость роста кристалла равна нулю. Связана с барьером нуклеации двумерных зародышей на грани |
| **Температура насыщения (tn)** | Температура, при которой раствор находится в термодинамическом равновесии с кристаллом. Выше tn — растворение, ниже tn — рост |
| **Температура начала растворения (te)** | Температура в точке isat — момент, когда интерферометр фиксирует начало растворения. Из-за мёртвой зоны te > tn |
| **Пересыщение (σ%)** | Относительное пересыщение: σ = 100·ln(C_n/C(T)), где C_n — равновесная концентрация при tn, C(T) — растворимость при T |
| **Кинетическая кривая R(σ)** | Зависимость скорости нормального роста грани кристалла от пересыщения |
| **LOESS** | LOcally Estimated Scatterplot Smoothing — метод локально-взвешенной регрессии |
| **Эталонные кривые** | Кривые R(σ) для чистого (Cfe=0) и загрязнённого (Cfe=16ppm) раствора при данной температуре |
| **Cfe** | Концентрация примеси железа Fe³⁺ в растворе (ppm). Основной загрязнитель KDP |
| **BCF** | Burton-Cabrera-Frank — модель спирального роста кристалла |
| **Призма/пирамида** | Грани кристалла KDP: призматические (боковые, {100}) и пирамидальные (торцевые, {101}) |

## 15. Примечания для реализации

### 15.1. Порядок вычислений

Алгоритм строго последовательный: каждый шаг зависит от предыдущих. Нельзя
параллелить шаги 3–7. Шаги 8 (эталоны) и 9 (Mat) можно выполнять параллельно с 7.

### 15.2. Числовые особенности

- `regress(x, y, 2)` в Mathcad возвращает вектор из 6 элементов: `[n, m, Err, a₀, a₁, a₂]`.
  Нужны элементы с индексами 3, 4, 5.
- `loess()` и `interp()` — встроенные функции Mathcad для LOESS-сглаживания.
  В Python аналог: `statsmodels.nonparametric.lowess` или `scipy` с ручной реализацией.
- `medsmooth(v, n)` — медианный фильтр с окном `n`. Аналог: `scipy.signal.medfilt`.
- `intercept(x, y)` и `slope(x, y)` — обычная линейная регрессия y = a + b·x.
- `csort(M, col)` — сортировка строк матрицы по столбцу `col`.
- `linterp(x, y, xi)` — линейная интерполяция.

### 15.3. Единицы измерения

- Температура: °C (всюду)
- Скорость роста: мм/день в графиках R(ΔT), мкм/мин в графиках R(σ%)
  (пересчёт: 1 мм/день ≈ 0.694 мкм/мин)
- Пересыщение: % (безразмерное × 100)
- Время: минуты (dtau в минутах)
- Фаза: радианы

### 15.4. Обработка зоны роста vs зоны растворения

В алгоритме обрабатывается **только зона растворения** (от isat1 до конца).
Зона роста используется только для расчёта среднего `y0`. Это важно:
кинетическая кривая строится по данным **растворения**, а не роста.

Физическое обоснование: кинетика растворения и роста связаны — при данном |σ|
скорость растворения ≈ скорость роста (при отсутствии гистерезиса).
Зона растворения удобнее, т.к. температура монотонно растёт → σ монотонно убывает.
