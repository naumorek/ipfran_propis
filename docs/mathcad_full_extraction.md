## 4. Нахождение экстремумов — функция `f` (скриншоты 1–2, 22)

### Алгоритм детекции экстремумов (zero-crossing, step=2):

```
f :=  | k ← 1     if S1₀ - y0 > 0
      | k ← -1    otherwise
      | j ← 0
      | for i ∈ 0,2..im1
      |   f_j ← i  if k·(S1_i - y0) < 0
      |   j ← j + 1  if k·(S1_i - y0) < 0
      |   k ← -k  if k·(S1_i - y0) < 0
      | f
```

**ВАЖНО**: Шаг итерации = 2 (`for i ∈ 0,2..im1`), НЕ 1!

### Вычисление y0s (baseline) — ТОЧНАЯ формула:
```
y0 := (1/(im1-1)) · Σ(i=0 to im1-1) S1_i       — среднее сигнала от 0 до im1-1
y0s := (1/(m-isat1)) · Σ(i=isat1 to m-1) S1_i   — среднее сигнала от isat1 до m-1
```

**ВАЖНО**: `y0` — среднее в зоне РОСТА (от 0 до im1). `y0s` — среднее в зоне РАСТВОРЕНИЯ (от isat1 до конца).

### Нахождение экстремумов mx (скриншот 22, жёлтый блок):

**Полный алгоритм `extrs` (из скриншота 22):**
```
extrs :=  | ma ← -10
          | mb ← -10
          | (k ← 1)   if (S1[isat1] - y0s) > 0
          | (k ← -1)  otherwise
          | j ← 0
          | for i ∈ isat1, isat1+2..m-1
          |   (ma ← i)  if √(S1_i - y0s)² > mb
          |   [mb ← √(S1_i - y0s)²]  if √(S1_i - y0s)² > mb
          |   (mx[j,0] ← ma)  if (S1_i - y0s)·k < 0
          |   (mx[j,1] ← S1[ma])  if (S1_i - y0s)·k < 0
          |   [j ← (j + 1)]  if (S1_i - y0s)·k < 0
          |   (mb ← -1)  if (S1_i - y0s)·k < 0
          |   (k ← -k)  if (S1_i - y0s)·k < 0
          |   (mx[j,0] ← ma)  if m-1-ma > 4
          |   (mx[j,1] ← mb)  if m-1-ma > 4
          | mx
```

**Ключевые отличия от зоны роста:**
- Зона роста: итерация от 0 до im1, baseline = y0
- Зона растворения: итерация от isat1 до m-1, baseline = y0s
- Шаг = 2 в обоих случаях!
- Экстремум фиксируется при смене знака `(S1_i - baseline)·k < 0`

## 6. Уточнение позиций экстремумов — квадратичная регрессия (скриншот 22–23)

### 6a. Функция `es` — регрессия degree=2:

```
es :=  | for j ∈ 0..ks
       |   (n ← |extrs[j+1,0] - extrs[j,0]|·k1)  if j < ks
       |   (n ← |extrs[j-1,0] - extrs[j,0]|·k1)  otherwise
       |   n ← floor(n)
       |   u ← extrs[j,0] - floor(n/2)  if j < 1
       |   (u ← extrs[j,0] - n)  otherwise
       |   (v ← extrs[j,0] + n)  if extrs[j,0] + n < m-1
       |   (v ← m-1)  otherwise
       |   for i ∈ u..v
       |     x[i-u,0] ← i - u
       |     x[i-u,1] ← S1_i
       |   y ← submatrix(x, 0, v-u, 0, 1)
       |   z ← regress(y⁽⁰⁾, y⁽¹⁾, 2)       — квадратичная регрессия
       |   q[j,0] ← -z₄/(2·z₅)               — вершина параболы: -a1/(2*a2)
       |   q[j,1] ← z₃ + z₄·q[j,0] + z₅·(q[j,0])²   — значение в вершине
       |   q[j,0] ← q[j,0] + u               — абсолютная позиция
       | (k ← -1)  if q[ks-1,1] < y0s
       | q
```

**ВАЖНО**: `z := regress(x, y, 2)` в Mathcad возвращает вектор, где:
- z₃ = a₀ (свободный член)
- z₄ = a₁ (линейный)
- z₅ = a₂ (квадратичный)

Вершина параболы: x_vertex = -z₄/(2·z₅), y_vertex = z₃ + z₄·x + z₅·x²

## 8. Разбиение синусоиды на части и фаза (d-step) — функция `y` (скриншоты 12–14, 28–29, s_1.jpg)

### 8a. Подготовка сетки:

```
g ← floor(isat1/d)                   — число d-step интервалов
```

### 8b. Полная функция `y` (построение фазы arcsin d-step):

```
y :=  | p ← rows(e) - 1              — e = refined extrema (es)
      | n ← 0
      | g ← floor(isat1/d)
      | for i ∈ 0..g+1
      |   a[i,0] ← i·d               — позиция на d-step сетке
      |   a[i,1] ← I(a[i,0])         — интерполированный сигнал I(x)
      |   a[i,3] ← t(a[i,0])         — интерполированная температура t(x)
      |   for j ∈ 0..p
      |     n ← j
      |     (break)  if e[j,0] > a[i,0]
      |
      |   a[i,2] ← -asin((a[i,1] - e[0,1])/(e[1,1] - e[0,1]))     if n < 1
      |
      |   a[i,2] ← π/2·(n-1) + asin((a[i,1] - e[n-1,1])/(e[n,1] - e[n-1,1]))   otherwise
      |
      |   s ← i
      |   (break)  if a[i,0] + d > e[p,0]
      |
      | for i ∈ s+1..g
      |   (break)  if s+1 > g
      |   a[i,0] ← i·d
      |   a[i,1] ← I(a[i,0])
      |   a[i,3] ← t(a[i,0])
      |   a[i,2] ← π/2·n + asin((a[i,1] - e[p,1])/(e[p-1,1] - e[p-1,1]))
      | a
```

**КРИТИЧЕСКИЙ МОМЕНТ — формула arcsin фазы:**

Между экстремумами n-1 и n:
```
phase[i] = π/2 · (n-1) + arcsin((S[i] - e[n-1,1]) / (e[n,1] - e[n-1,1]))
```

Перед первым экстремумом (n < 1):
```
phase[i] = -arcsin((S[i] - e[0,1]) / (e[1,1] - e[0,1]))
```

После последнего экстремума:
```
phase[i] = π/2 · n + arcsin((S[i] - e[p,1]) / (e[p-1,1] - e[p,1]))
```

**Замечание**: знаменатель МЕНЯЕТ ЗНАК после последнего экстремума: `e[p-1,1] - e[p,1]` вместо `e[p,1] - e[p-1,1]`.

## 10. Вычисление L1 и скорости zs-формулой (скриншоты 12–14, 28–29)

### 10a. Определение границ u, v (зона роста):

```
u :=  | for i ∈ 0..rows(y) - 1
      |   u ← i
      |   break  if y[i,0] > im
      | u - 1

v :=  | for i ∈ u..rows(y) - 1
      |   v ← i
      |   break  if y[i,0] > isat
      | v - i - 1
```

### 10b. Вычисление L1:

```
g := u..v
l2[p-u] := y[g,3]                     — температура в зоне u..v
P[p-u] := y[g,2]                      — фаза в зоне u..v

Q := slope(l2, p)                     — наклон: фаза vs температура
L1 := -Q / (π · co₁)                 — L1 = −Q/(π·co₁)

x0 := co₀ - co₁·y[0,3]              — оптический коэфф. при начальной температуре

L0 := L1 · (co₀ - co₁·y[0,3]) / x0 · (y[u,2] - y[0,2]) / (π · x0)
L0 = 3.3364 × 10³
```

### 10c. Функция L — оптическая толщина:

```
L :=  | for i ∈ 0..rows(y) - 1
      |   x ← co₀ - co₁·y[i,3]
      |   L_i ← (y[i,2] / (π·x) + L0 · x0/x - L0) · nn
      | L
```

### 10d. Формула zs — СКОРОСТЬ РОСТА (КЛЮЧЕВАЯ ФОРМУЛА):

```
zs :=  | for i ∈ 0..n-1              — n = rows(y) - 1
       |   z[i,0] ← (ys[i,0] + ys[i+1,0]) / 2                — средняя позиция
       |   y1 ← floor(ys[i,0])                                 — целая часть позиции
       |   y2 ← floor(ys[i+1,0])                               — целая часть след. позиции
       |   z[i,2] ← 0.5·(a[y2,4] + a[y1,4]) - 273.15         — средняя T в °C
       |   x0 ← co₀ - co₁·(a[m-1,4] - 273.15)                — опт. коэфф. при T_ref
       |   x1 ← co₀ - co₁·(a[y1,4] - 273.15)                 — опт. коэфф. при T1
       |   x2 ← co₀ - co₁·(a[y2,4] - 273.15)                 — опт. коэфф. при T2
       |
       |   z[i,1] ← [L1·x0·(1/x2 - 1/x1) + 1/π·(ys[i+1,1]/x2 - ys[i,1]/x1)]
       |             · 30 / ((ys[i+1,0] - ys[i,0])·ww)
       | z
```

**КЛЮЧЕВАЯ ФОРМУЛА скорости:**
```
R[i] = [L1·x0·(1/x2 - 1/x1) + (1/π)·(phase[i+1]/x2 - phase[i]/x1)] · 30 / (Δpos · ww)
```

Где:
- `L1 = -Q/(π·co₁)` — нормировочный коэффициент из наклона фаза vs температура
- `x0` = co₀ - co₁·T_ref — оптический коэфф. при референсной температуре
- `x1, x2` = co₀ - co₁·T₁,₂ — опт. коэфф. при температурах на концах интервала
- `phase[i], phase[i+1]` — фаза arcsin на d-step сетке (= ys[i,1])
- `Δpos = ys[i+1,0] - ys[i,0]` = d (шаг)
- `30` — пересчёт в мм/день (30 минут/день / dtau)
- `ww` — весовой множитель (обычно 1)

### 10e. Пересыщение zs[in,3]:

```
in := 0..rows(zs) - 1
zs[in,3] := 100·ln(Cn / (co₂ + co₃·zs[in,2] + co₄·(zs[in,2])²))
```

Это σ в процентах: `σ = 100·ln(Cn/Cm)`, где:
- `Cn = co₂ + co₃·te + co₄·te²` — растворимость при te
- `Cm = co₂ + co₃·T + co₄·T²` — растворимость при текущей T

       |   ys[j,1] ← π/2·(s + 1 - i)  otherwise
       |
       |   j ← j + 1
       |
       | for i ∈ 0..ss                         — ss = число полупериодов
       |   for r ∈ 0..es[i,2] - 1              — для каждой позиции внутри полупериода
       |     ys[j,0] ← es[ss+1,0]
       |     ys[j,1] ← π/2·(s - ss)
       |   j ← j + 1
       |
       | ys[j,0] ← es[ss+1,0]
       | ys[j,1] ← π/2·(s - ss)

Ось Y: `R(mm/day)` = (z)[i,1], 1.441·F1(z[i,1]), (1.441·Si)[j,1], (1.441·Si1)[j,1], (1.441·Z)[k,1], (1.441·ZFe)[k,1]

**Множитель 1.441**: пересчёт из мкм/мин в мм/день (1 мкм/мин × 60 мин × 24 час / 1000 мкм/мм = 1.44 мм/день).

### Формула пересчёта σ → dT на графике:

```
dT = (tn/100) · σ
```

Где σ — пересыщение в %.

**ВАЖНО**: На оси X графика стоит `dT(C)`, но данные хранятся в σ(%). Пересчёт: `dT ≈ tn/100 · σ`, что для tn≈49: `dT ≈ 0.49 · σ`.

## 27. Ранее неизвестные/уточнённые детали

1. **tn содержит коррекцию**: `tn = a[isat,4] - 273.15 - 0.01·(te - 25)` — ранее считали что tn вводится вручную
2. **y0 vs y0s**: baseline для зоны роста (y0) и растворения (y0s) — РАЗНЫЕ значения, вычисляются из разных диапазонов
3. **Шаг экстремумов = 2**: не 1, а именно 2 (`for i ∈ 0,2..im1`)
4. **Множитель 1.441**: пересчёт мкм/мин → мм/день на графике
5. **Формула пересчёта σ → dT**: `dT = (tn/100)·σ`, а не просто tn - T
6. **Коэффициенты co₀, co₁ домножаются на gran1/gran2**: `co₀ = 0.06446·gran1`, `co₁ = 1.92×10⁻⁵·gran2`
7. **Матрицы a/a1 vs b/b1**: `a` = кислый (Acid=1), `b` = нейтральный (Acid=0). Выбор: `a := if(Acid < 1, b, a)`
8. **s2 = -Q1/Q2**: из log-log регрессии R vs (σ - s₁), НЕ из основного фита
9. **Q матрица**: фильтрация: только точки с 0 < R < 0.3
10. **dmin=-0.2, dmax=-0.0**: границы выборки для линейной регрессии zs1 в зоне растворения
