## 1. Проводимость → пересыщение σ(t)

### Принцип

Электропроводность раствора KH₂PO₄ зависит от концентрации C и температуры T. Измеряя κ(t) и T(t) одновременно, вычисляем C(t) по калибровочной поверхности, затем σ(t) = (C(t) − C_sat(T)) / C_sat(T).

Метод описан в: Belruss et al. (1984) J. Crystal Growth 68, 119. Использовался в LLNL для кристаллов NIF.

### Электроды

**Материал:** платинированная платина (лучше) или нержавеющая сталь AISI 316L (проще).

**Конструкция:**
- Две пластины ~10×10 мм, расстояние ~10 мм (константа ячейки K ≈ 1 см⁻¹)
- Закрепить на фторопластовом (PTFE) держателе — химически инертен в KH₂PO₄
- Провода — тефлоновая изоляция, выход через гермоввод (сальник PG9/PG11)

**Платинирование** (если доступна платинохлористоводородная кислота):
```
Раствор: 1% H₂PtCl₆ + 0.01% Pb(CH₃COO)₂ в воде
Режим: 10–30 мА/см², 5–10 мин, переменная полярность каждые 30 сек
Результат: чёрный налёт Pt → увеличивает эффективную площадь,
           снижает поляризацию, стабилизирует показания
```

Если платинирование недоступно — нержавейка работает, но дрейф больше. Периодическая очистка (раз в неделю) разбавленной HCl.

### Электроника

**Важно:** измерять на **переменном токе** (≥1 кГц), чтобы избежать электролиза и поляризации электродов.

**Схема (простая):**
```
Генератор 1 кГц, синус, 0.5–1 V
    │
    ├── Электроды в растворе (сопротивление R_x)
    │
    ├── Эталонный резистор R_ref (100–1000 Ом)
    │
    └── Детектор амплитуды → АЦП

R_x = R_ref × (V_gen / V_ref − 1)
κ = K_cell / R_x   [мСм/см]
```

**Генератор 1 кГц:**
- Вариант A: лабораторный генератор (если есть)
- Вариант B: Arduino + ЦАП MCP4725 (синус из таблицы, 1 кГц)
- Вариант C: таймер NE555 (прямоугольник 1 кГц) + RC фильтр → квази-синус

**Детектор амплитуды:**
- Вариант A: прецизионный выпрямитель на ОУ (AD620 или LM358) + RC фильтр (τ = 100 мс)
- Вариант B: программный — АЦП на высокой частоте (ADS1115 на 860 SPS), оцифровать синус, амплитуду вычислить в МК

**Готовый модуль (проще всего):**
- DFRobot Gravity: Analog EC Sensor Kit (~30 EUR на Ali)
- Включает: датчик, усилитель, плата обработки, BNC разъём
- Или: Atlas Scientific EZO-EC (~80 EUR) — калиброванный, I2C выход

### Калибровка (главная работа, 2–3 дня)

**Приготовление растворов:**
```
Концентрации: 20, 25, 28, 30, 32, 34, 36, 38, 40 г KDP / 100 г H₂O
(9 точек, покрывают рабочий диапазон 30–40 г/100г при 40–60°C)
```

Использовать аналитические весы (±0.01 г) и дистиллированную воду.

**Измерения:**
```
Для каждого раствора — измерить κ при T = 35, 40, 45, 50, 55, 60°C
Термостат (водяная баня) + перемешивание до стабилизации T (±0.1°C)
Записать: (C_i, T_j, κ_ij)
Итого: 9 × 6 = 54 точки
```

**Построение калибровочной модели (Python):**
```python
# Полиномиальная поверхность 2-го порядка
# C = a0 + a1*κ + a2*T + a3*κ² + a4*T² + a5*κ*T
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression

poly = PolynomialFeatures(degree=2)
X = poly.fit_transform(np.column_stack([kappa_array, T_array]))
model = LinearRegression().fit(X, C_array)
# R² > 0.999 ожидается для бинарной системы
```

**Проверка:** приготовить 3 контрольных раствора (не из калибровки), измерить κ и T, предсказать C, сравнить с истинным.

**Ожидаемая точность DIY:** ±0.1–0.3 г/100г → ±0.3–1.0% пересыщения. Хуже промышленного (±0.05 г), но достаточно для первых экспериментов — мёртвая зона (σ_dead ≈ 0.3%) на границе разрешения, зона роста (σ > 1%) — уверенно.

## 2. Температура T(t) с точностью ±0.03°C

### Датчик

**Pt100 RTD, класс 1/3 DIN или AA:**
- Ali/eBay: "Pt100 class AA thin film" — 15–30 EUR/шт.
- Диаметр гильзы 3–6 мм, длина 30–100 мм, нержавейка
- Заказать 3 шт.: 1 в объёме раствора, 1 у кристалла, 1 запасной

**4-проводное подключение** (обязательно для точности):
```
Pt100 имеет R ≈ 100 Ом при 0°C, α ≈ 0.385 Ом/°C
При 50°C: R ≈ 119.4 Ом
Изменение ±0.03°C → ΔR ≈ ±0.012 Ом
Сопротивление проводов (2 м) ≈ 0.5 Ом → вносит ошибку ≈ 1.3°C!
4-проводная схема компенсирует это полностью.
```

### Электроника

**Вариант A: мультиметр (если есть)**
- Keithley 2000, Agilent 34401A или аналог в режиме 4-wire Ω
- Разрешение: 0.001 Ом → ±0.003°C
- Лучший вариант, но мультиметр привязан

**Вариант B: ADS1115 + токовый источник (DIY)**
```
Токовый источник 1 мА (LM334 или REF200):
  V_pt100 = I × R_pt100 = 0.001 × 119.4 = 119.4 мВ при 50°C

ADS1115 (16 бит, PGA ×8, диапазон ±0.512V):
  Разрешение = 1.024 / 65536 = 15.6 мкВ
  Температурное разрешение = 15.6 мкВ / (0.001 × 0.385) = 0.04°C

С усредняем 100 отсчётов: ±0.004°C — достаточно.
```

**Вариант C: MAX31865 — готовый модуль RTD→SPI (~5 EUR на Ali)**
- Специализированная микросхема для Pt100
- 15-бит разрешение, встроенный токовый источник, SPI интерфейс
- Разрешение: ±0.03°C из коробки
- Подключение к Arduino/ESP32 — 3 провода

**Рекомендация:** MAX31865 — самый простой путь. 5 EUR за модуль, ±0.03°C, никакой аналоговой разработки.

### Калибровка

**Ледяная точка (0.000°C):**
```
Термос + мелкий лёд из дистиллированной воды + дистиллированная вода
Перемешать, подождать 5 мин → T = 0.000 ± 0.005°C
Записать показание → offset
```

**Вторая точка:** сравнение с эталонным ртутным термометром при 50°C (водяная баня).

### Обработка (Python + OpenCV)

```python
import cv2
import numpy as np
from datetime import datetime

cap = cv2.VideoCapture(0)

# Калибровка: пиксели → мм (линейка в кадре, один раз)
SCALE = 0.15  # мм/пиксель (пример)

while True:
    ret, frame = cap.read()
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

    # Контраст + пороговая бинаризация
    _, thresh = cv2.threshold(gray, 100, 255, cv2.THRESH_BINARY)

    # Контуры кристалла
    contours, _ = cv2.findContours(thresh, cv2.RETR_EXTERNAL,
                                    cv2.CHAIN_APPROX_SIMPLE)
    if contours:
        # Наибольший контур = кристалл
        c = max(contours, key=cv2.contourArea)
        rect = cv2.minAreaRect(c)  # (center, (w, h), angle)
        w_mm = rect[1][0] * SCALE
        h_mm = rect[1][1] * SCALE

        timestamp = datetime.now().isoformat()
        print(f"{timestamp}, width={w_mm:.2f} mm, height={h_mm:.2f} mm")
        # → записать в CSV

    time.sleep(600)  # каждые 10 мин
```

**Для более точного определения граней:** вместо `minAreaRect` — детектор линий (Hough transform) + фит граней призмы и пирамиды. Или ML-сегментация (обучить на 50–100 размеченных кадрах).

### Калибровка

Поместить линейку (или калибровочную мишень с известными размерами) в поле зрения камеры на расстоянии кристалла. Определить SCALE (мм/пиксель). Пересчитывать при изменении расстояния/фокуса.

### Калибровка при рабочей T

```
1. Буферы pH 4.01 и 7.00 нагреть до рабочей T (50°C) в водяной бане
2. Калибровка 2 точки: slope и offset при 50°C
3. Дополнительно: измерить pH раствора KDP при 25°C и 50°C
   → определить ΔpH/ΔT для вашего раствора (≈ -0.004 pH/°C)
```

**Важно:** стандартные буферы имеют табличные значения pH при разных T (указано на упаковке). Использовать значения при 50°C, не при 25°C.

## 5. Теневой метод (shadowgraph) — визуализация потоков

### Что нужно

Практически ничего нового:
- Точечный источник света: яркий LED (1–3 Вт) + диафрагма (отверстие 1–2 мм в алюминиевой фольге)
- Расстояние: LED → кристаллизатор → экран/камера, ~1–2 м суммарно
- Экран: белая бумага на стене, или камера (та же что для R(t))

### Что увидим

Конвективные потоки (plumes) вокруг кристалла создают градиенты концентрации → градиенты показателя преломления → тени. Видно:
- Обеднённый слой раствора, стекающий с граней
- Конвективные ячейки
- Асимметрию потока (верх/низ, лево/право)

Это качественная картина, но очень информативная для понимания массопереноса и валидации CFD.

**Литература:** Srinivasan et al. (2004) J. Crystal Growth 264, 192 — прямое сравнение интерферометрии, шлирен и shadowgraph для KDP.

## 6. Сбор данных — единая система

### ESP32 + модули (рекомендация)

```
ESP32 DevKit (~10 EUR)
  ├── MAX31865 × 2 (Pt100 объём + Pt100 у кристалла)  → SPI
  ├── ADS1115 (проводимость + pH аналог)               → I2C
  ├── SD-карта модуль                                    → SPI
  └── WiFi → отправка данных на сервер / MQTT

Питание: USB 5V (power bank или адаптер)
Корпус: пластиковый, IP65 если нужна защита от брызг
```

### Софт (MicroPython)

```python
# main.py на ESP32
from machine import Pin, SPI, I2C
from max31865 import MAX31865
from ads1115 import ADS1115
import time, sdcard, os

# Инициализация
spi = SPI(1, baudrate=1000000)
rtd1 = MAX31865(spi, Pin(5))   # Pt100 #1
rtd2 = MAX31865(spi, Pin(17))  # Pt100 #2
i2c = I2C(0, scl=Pin(22), sda=Pin(21))
adc = ADS1115(i2c)

# SD карта
sd = sdcard.SDCard(SPI(2), Pin(15))
os.mount(sd, '/sd')

while True:
    T1 = rtd1.temperature()  # объём раствора
    T2 = rtd2.temperature()  # у кристалла
    kappa = adc.read(0)       # проводимость (сырое)
    ph_raw = adc.read(1)      # pH (сырое)

    line = f"{time.time()},{T1:.3f},{T2:.3f},{kappa},{ph_raw}\n"
    with open('/sd/log.csv', 'a') as f:
        f.write(line)

    time.sleep(10)  # каждые 10 секунд
```

## 8. Порядок работ

### Неделя 1: электроника и калибровка
1. Заказать компоненты (Ali, 1–2 недели доставка, или локально)
2. Собрать ESP32 + MAX31865 + ADS1115 на макетной плате
3. Изготовить электроды проводимости (нерж./Pt + PTFE держатель)
4. Калибровка Pt100 (ледяная точка + 50°C)
5. Калибровка проводимости (54 точки: 9 концентраций × 6 температур)

### Неделя 2: установка и тест
6. Установить датчики в кристаллизатор (гермовводы, кабели)
7. Настроить камеру + подсветку + скрипт OpenCV
8. Тестовый прогон: 24 часа записи → проверить стабильность, дрейф
9. Shadowgraph — фото/видео конвективных потоков
10. Первые данные σ(t), T(t), R(t) → первый R(σ) из кристаллизатора

### Далее: итерация
11. Сравнить R(σ)_кристаллизатор с R(σ)_стенд
12. Определить β_крист / β_стенд → оценка массопереноса
13. Мониторинг σ_dead(t) по дням → деградация раствора
