# Справочник по Mathcad 2000 Professional

## Формат файлов MCD (версия 3.09)

### Общие сведения

- **Сигнатура файла:** `.MCAD 309000000` (hex: `2E4D 4341 4420 3330 3930 3030 3030 30`)
- **Формат:** OLE (Object Linking and Embedding) — составной бинарный документ
- **Расширение:** `.mcd`
- **Совместимость:** Mathcad 2000 Professional, частично совместим с Mathcad 11–15

### Структура документа

MCD-файл содержит следующие OLE-объекты:

| Объект | Назначение |
|--------|-----------|
| `docDocument` | Главный контейнер документа |
| `mcObject` | Фреймворк объектов Mathcad |
| `mcDocumentObjectState` | Состояние документа |
| `mcPageModel` | Настройки макета страницы |
| `mcHeaderFooter` | Колонтитулы |
| `eqRegion` | Области формул (уравнений) |
| `TextRegion` | Текстовые области |
| `docRegion` | Области документа |
| `d2_graph_format` | Форматирование 2D-графиков |
| `trace2D` | Данные трассировки графиков |
| `axisFormat` | Форматирование осей |
| `dim_format` | Форматирование размерностей |
| `NumericalFormat` | Числовые форматы |

### Шрифты в документах

- **Times New Roman Cyrillic** — основной текст
- **Courier** — формулы и код
- **Arial** — заголовки
- **Symbol** — математические символы

## Синтаксис Mathcad 2000

### Типы операторов

| Оператор | Синтаксис | Описание |
|----------|-----------|----------|
| Присваивание | `x := 5` | Определение переменной |
| Вычисление | `x =` | Вывод значения |
| Глобальное присваивание | `x ≡ 5` | Определение, видимое во всём документе |
| Диапазон | `i := 0..n-1` или `i := 0, 1..n-1` | Определение переменной диапазона |

### Индексация

- **Массивы начинаются с индекса 0** (по умолчанию, `ORIGIN = 0`)
- Обращение к элементу: `a[i]` или `a[i,j]` для матриц
- Обращение к столбцу матрицы: `M^<j>` (верхний индекс в угловых скобках)

## Встроенные функции, используемые в алгоритме

### Ввод/вывод данных

#### `READPRN(file)`
- **Назначение:** Чтение данных из текстового файла (PRN)
- **Вход:** Имя файла (строка), файл должен быть в текущей директории
- **Выход:** Матрица данных (числовая)
- **Формат файла:** Столбцы разделены пробелами/табуляцией, строки — переносами
- **Пример:** `a := READPRN("__030326_5.prn")` → матрица ~65000×7

### Статистическая обработка

#### `medsmooth(v, n)`
- **Назначение:** Медианное сглаживание вектора
- **Вход:** `v` — вектор данных, `n` — ширина окна (нечётное число)
- **Выход:** Сглаженный вектор той же длины
- **Особенности:** Устойчиво к выбросам, сохраняет резкие перепады

#### `loess(vx, vy, span)`
- **Назначение:** Локальная регрессия (LOESS/LOWESS)
- **Вход:** `vx` — вектор X, `vy` — вектор Y, `span` — параметр сглаживания (0 < span ≤ 1)
- **Выход:** Вектор коэффициентов для использования с `interp()`
- **Использование:** `vs := loess(x, y, 0.15)`, затем `interp(vs, x, y, xi)`

#### `regress(vx, vy, n)`
- **Назначение:** Полиномиальная регрессия
- **Вход:** `vx` — вектор X, `vy` — вектор Y, `n` — степень полинома
- **Выход:** Вектор коэффициентов

### Интерполяция

#### `linterp(vx, vy, x)`
- **Назначение:** Линейная интерполяция
- **Вход:** `vx` — отсортированный вектор X, `vy` — вектор Y, `x` — точка интерполяции
- **Выход:** Интерполированное значение

#### `interp(vs, vx, vy, x)`
- **Назначение:** Интерполяция по заранее вычисленным коэффициентам
- **Вход:** `vs` — вектор от `loess`/`regress`, `vx`, `vy` — данные, `x` — точка
- **Выход:** Значение в точке x

### Работа с матрицами и векторами

#### `submatrix(M, r1, r2, c1, c2)`
- **Назначение:** Извлечение подматрицы
- **Вход:** `M` — исходная матрица, `r1..r2` — диапазон строк, `c1..c2` — диапазон столбцов
- **Выход:** Подматрица M[r1:r2, c1:c2]

#### `csort(M, n)`
- **Назначение:** Сортировка матрицы по столбцу n
- **Вход:** `M` — матрица, `n` — номер столбца для сортировки
- **Выход:** Матрица, строки которой отсортированы по значениям столбца n

#### `stack(A, B)`
- **Назначение:** Вертикальная конкатенация матриц/векторов
- **Вход:** Две матрицы с одинаковым числом столбцов
- **Выход:** Объединённая матрица

#### `reverse(v)`
- **Назначение:** Обращение порядка элементов вектора
- **Выход:** Вектор с элементами в обратном порядке

#### `rows(M)`, `cols(M)`
- **Назначение:** Число строк / столбцов матрицы

#### `last(v)`
- **Назначение:** Индекс последнего элемента вектора

#### `sort(v)`
- **Назначение:** Сортировка вектора по возрастанию

### Математические функции

#### `slope(vx, vy)`
- **Назначение:** Наклон линии линейной регрессии (МНК)

#### `intercept(vx, vy)`
- **Назначение:** Свободный член линейной регрессии (МНК)

#### `ceil(x)`, `floor(x)`
- **Назначение:** Округление вверх / вниз до целого

#### `asin(x)`
- **Назначение:** Арксинус

#### `ln(x)`, `exp(x)`
- **Назначение:** Натуральный логарифм / экспонента

#### `max(v)`, `min(v)`
- **Назначение:** Максимум / минимум вектора

## Программные конструкции

### Программные блоки (Programming)

Mathcad 2000 поддерживает программные блоки с вертикальной чертой слева:

```
f(x) := | result ← 0
         | for i ∈ 0..n-1
         |   result ← result + x_i
         | result
```

### Поддерживаемые конструкции

| Конструкция | Описание |
|-------------|----------|
| `←` | Локальное присваивание внутри программного блока |
| `for i ∈ range` | Цикл по диапазону |
| `while condition` | Цикл с условием |
| `if condition` | Условный оператор |
| `otherwise` | Ветка else |
| `break` | Выход из цикла |
| `continue` | Следующая итерация |
| `return` | Возврат значения |

### Пример из алгоритма — детекция экстремумов:
```
extrs := | s ← -1
          | n ← 0
          | for i ∈ 0..length-1
          |   if |S2_i - y0| > a
          |     l ← S2_i
          |     n ← i
          |     s ← |S2_i - y0|
          |   s ← -1
          | ...
```

## Ограничения Mathcad 2000

| Ограничение | Значение |
|-------------|----------|
| Максимальный размер матрицы | ~8 млн элементов |
| Точность вычислений | float64 (IEEE 754 double) |
| Максимальная вложенность циклов | ~20 уровней |
| Символьные вычисления | Ограниченные (через Maple kernel) |
| Формат файлов данных | Только PRN (текстовые, разделённые пробелами) |
| Графики | Только 2D и 3D, без интерактивности |
| Размер файла MCD | Типично 150–170 КБ для алгоритма обработки |

### Особенности, важные для миграции

1. **READPRN** не поддерживает CSV, только пробельные разделители
2. **Программные блоки** ограничены — нет массивов переменной длины, нет рекурсии
3. **Графики** привязаны к областям документа — нет экспорта данных из графика
4. **Оптимизация** — нет встроенных алгоритмов оптимизации (Minimize/Maximize появились позже); в алгоритме используется ручной перебор
5. **Обработка ошибок** — отсутствует; ошибочные данные приводят к остановке вычислений
6. **Единицы измерения** — поддерживаются, но в данном алгоритме не используются (все величины безразмерные)
