# Исследование: Dead Zone в Modern (Hilbert) Pipeline

## Проблема

Modern pipeline (Hilbert instantaneous frequency) даёт ненулевую скорость роста R при dT=0 (температура насыщения). Физически — скорость должна быть нулевой.

Classic pipeline тоже иногда имеет ненулевой baseline при dT≈0, но реже — arcsin-фаза чаще collapse к нулю.

---

## Физика мёртвой зоны

**Мёртвая зона (dead zone)** — область вблизи температуры насыщения tn, где скорость роста кристалла равна нулю. Причина — **механизм Кабреры-Вермильи**: примесные частицы (особенно Fe³⁺) адсорбируются на террасах кристалла между ступенями и блокируют их движение.

- Если расстояние между стопперами < 2r₀ (критический радиус), ступень заблокирована
- Рост возобновляется только при σ > σ_d (пересыщение больше порога)
- При увеличении σ рост идёт через движение **макроступеней** (пучков элементарных ступеней)

### Ширина мёртвой зоны

```
Td := te - tm                    — мёртвая зона в °C (типично 0.2–0.4°C)
Sigm := 100·ln(Cn/Cm)           — мёртвая зона в % пересыщения (типично 0.3–0.5%)
σ_d = 2Ω·α·Cᵢ^(1/2) / kT      — модель Кабреры-Вермильи
```

Зависимость от Fe³⁺:

| Концентрация Fe³⁺ | Эффект |
|---|---|
| <1 ppm | Нормальный рост |
| ~5 ppm | Минимальное влияние |
| ~16 ppm | Значительное замедление (эталонная кривая) |
| ~30 ppm | "Суперступени" до 1500 нм |
| ~100 ppm | Сильная кинетическая блокировка |

## Почему Mathcad (arcsin) естественно обнуляет скорость

### Arcsin phase collapse

Mathcad строит фазу между экстремумами через arcsin:
```
phase[i] = π/2 · (n-1) + arcsin((S[i] - e[n-1,1]) / (e[n,1] - e[n-1,1]))
```

В мёртвой зоне:
1. Амплитуда осцилляций → 0 (сигнал flat)
2. Знаменатель `(e[n,1] - e[n-1,1])` → 0
3. Фаза фиксируется на последнем значении (constant)
4. d(phase)/dT → 0 → **rate → 0 естественно**

### Grid search cutoff

После вычисления raw rates, Mathcad явно отсекает:
```
break if VY_k > 0.01   ← пропуск rates < 0.01 mm/day
```

Затем LOESS сглаживание только на "реальных" данных. Экстраполяция к dT=0 даёт нулевую скорость.

## Почему Hilbert НЕ обнуляет скорость в dead zone

### Математическая причина

1. `analytic_signal = signal + j·hilbert(signal)`
2. `inst_freq = d(phase)/dt`, где `phase = arctan2(imag, real)`
3. Около tn: амплитуда → 0, сигнал = шум
4. При нулевой амплитуде: любой шум → **произвольные фазовые скачки** → произвольная inst_freq
5. Savitzky-Golay размазывает частоту через границу tn

### Принципиальное отличие от arcsin

- Arcsin работает с **дискретными парами экстремумов** (robust)
- Hilbert даёт **непрерывную фазу** (fragile при малой амплитуде)
- Arcsin имеет **встроенную сингулярность** (знаменатель → 0), Hilbert — нет

## Варианты решения

### 1. Envelope masking (быстрый fix)

Обнулить rate где envelope < порога:
```python
analytic = scipy_hilbert(preprocessed)
envelope = np.abs(analytic)
dead_mask = envelope < 0.15 * np.max(envelope)
rate_continuous[dead_mask] = 0
```

**Плюсы:** Простой, 3 строки кода. Signal-driven — работает для любого кристалла/грани/T.
**Минусы:** Порог 15% произвольный, может обрезать данные в зоне перехода.

### 2. Temperature cutoff по Td (быстрый fix)

Обнулить rate при dT < Td:
```python
dT = tn - temp_c
rate_continuous[dT < Td] = 0  # Td уже вычислен в pipeline
```

**Плюсы:** Физически мотивирован (Td = ширина мёртвой зоны). Самокалибруется по im/isat каждого эксперимента.
**Минусы:** Td — грубая оценка; жёсткий порог создаёт разрыв кривой.

### 3. CWT Ridge вместо Hilbert (рекомендуется)

Заменить Hilbert на wavelet ridge extraction:
```python
from ssqueezepy import cwt, extract_ridges, Wavelet

wavelet = Wavelet('gmw')  # Generalized Morse Wavelet
Wx, scales = cwt(signal, wavelet)
ridge_idxs, *_ = extract_ridges(Wx, scales, penalty=2.0, n_ridges=1, bw=25)
inst_freq = freqs_axis[ridge_idxs[0]]
```

**Механизм:** Wavelet ridge автоматически ослабевает при малой амплитуде сигнала. Не требует явного порога — dead zone suppression встроено в метод.

**Плюсы:** Robust, без произвольных параметров, лучше адаптируется к нестационарным частотам.
**Минусы:** Новая зависимость (`ssqueezepy`), потенциально медленнее.

### 4. PLL (Phase-Locked Loop)

Петля фазовой автоподстройки:
```python

    # Петля отслеживает фазу пока есть сигнал

```

**Плюсы:** Лучшая обработка dead zone ("Excellent" в таблице). Нет скачков при потере сигнала.
**Минусы:** Требует настройки bandwidth и центральной частоты.

### 5. Mathcad-style фильтр R > 0.01 в LOESS (быстрый fix для отображения)

Фильтровать мелкие скорости перед LOESS — аналог Mathcad grid search cutoff:
```python
mask = (dT_dense > 0) & (R_dense > 0.01)
```

**Плюсы:** Точное воспроизведение поведения Mathcad. Работает для обоих pipeline.
**Минусы:** Не исправляет сами данные, только отображение.

## Анализ модели Кабреры-Вермильи (CV) для практического применения

### Формула CV

```
σ_d = 2Ω·α·Cᵢ^(1/2) / kT
```

Где:
- `Ω` — молекулярный объём (~9.7×10⁻²⁹ м³ для KDP)
- `α` — свободная энергия ступени: ~62 pJ/m для {100}, ~27 pJ/m для {101}
- `Cᵢ` — **поверхностная** плотность адсорбированных примесей (шт/м²)
- `kT` — тепловая энергия

Экспериментально подтверждено (Rashkovich, Kronsky, J. Cryst. Growth 1997; Land, De Yoreo, Nature 1999):
- σ_d ∝ √[Fe³⁺] для {100} граней KDP
- При ~16 ppm Fe³⁺: σ_d ≈ 0.3–0.5% (совпадает с нашим Sigm)
- Грань {100} (призма) чувствительнее к Fe³⁺, чем {101} (пирамида)
- Dead zone уменьшается с ростом температуры

### Связь объёмной и поверхностной концентрации

Модель Kubota-Mullin: покрытие через Langmuir-адсорбцию:
```
θ = K·c / (1 + K·c)
Cᵢ = θ · Cᵢ_max
```
Константа K зависит от T, pH, грани. Для KDP + Fe³⁺ **не опубликована** в числовом виде.

### Почему CV модель НЕ практична как инструмент

Наша система охватывает широкий диапазон условий:
- **Кристаллы:** KDP и DKDP (разные Ω и α)
- **Грани:** {100} призма и {101} пирамида (разная адсорбция, step energy)
- **Температура:** 10–70°C (kT меняется, α и K сильно зависят от T)
- **Добавки:** ортофосфорная кислота (меняет pH → меняет адсорбцию Fe³⁺)

CV модель требует **отдельную калибровку** для каждой комбинации:
кристалл × грань × кислота × температурный диапазон.

| Параметр | Известен? | Комментарий |
|---|---|---|
| Ω (молекулярный объём) | Да | KDP: ~9.7×10⁻²⁹, DKDP: чуть больше |
| α (step edge energy) | Частично | {100}: ~62 pJ/m, {101}: ~27 pJ/m (только для KDP, T≈25°C) |
| K (конст. адсорбции) | **Нет** | Не опубликована для KDP + Fe³⁺ ни при каких условиях |
| Cᵢ_max | **Нет** | Максимальная поверхностная плотность неизвестна |
| [Fe³⁺] в растворе | Не всегда | Известна только для эталонных растворов (0, 16 ppm) |

**Вывод:** CV модель объясняет физику dead zone, но не даёт числовой предсказательной силы для нашего инструмента. Мы и так измеряем Td/Sigm эмпирически в каждом эксперименте — это надёжнее любой модельной оценки.

Единственное полезное следствие: эмпирическая зависимость `σ_d ≈ A · √[Fe³⁺]` для оценки качества раствора. Но это научный результат, не улучшение pipeline.

## Рекомендация

### Для рабочего инструмента (все условия: KDP/DKDP, {100}/{101}, 10–70°C, ±кислота)

Практичный fix dead zone должен быть **самокалибрующимся из данных** каждого эксперимента, без справочников и модельных параметров.

Три подхода, работающие для любых условий (dead zone всегда проявляется одинаково в сигнале: амплитуда осцилляций → 0):

1. **Envelope masking** — signal-driven, 3 строки кода
2. **Td cutoff** — самокалибруется из im/isat
3. **CWT Ridge** — wavelet ridge ослабевает при малой амплитуде автоматически

**Краткосрочно:** Вариант 5 (фильтр R > 0.01 в LOESS, аналог Mathcad) + вариант 1 или 2 для данных.

**Стратегически:** Переход на CWT Ridge (вариант 3) — решает dead zone без костылей и даёт лучшее разрешение для нестационарных частот.

## Также выявлено: Classic dead zone baseline

Classic pipeline тоже иногда имеет ненулевой R при dT≈0 (видно на 030326_6_3, 040326_5_3). Причина: arcsin-фаза на шумном flat-сигнале не всегда collapse к нулю. Mathcad решает это grid search cutoff (rates < 0.01 отсекаются до LOESS). Аналогичный фильтр `R > 0.01` в `get_f1_curve()` решит проблему для обоих pipeline на уровне отображения.
