## 1. Установка и запуск

### 1.1 Требования

- **Python 3.10 или выше** (проверить: `python3 --version`)
- Операционная система: Windows, Linux или macOS
- Для GUI: поддержка графического дисплея (X11, Wayland или Windows Desktop)

### 1.2 Установка

Откройте терминал в папке проекта и выполните:

```bash
# Создание виртуального окружения
python3 -m venv .venv

# Активация (Linux/macOS)
source .venv/bin/activate

# Активация (Windows PowerShell)
.venv\Scripts\Activate.ps1

# Активация (Windows CMD)
.venv\Scripts\activate.bat

# Установка зависимостей
pip install -r propis_app/requirements.txt
```

### 1.3 Запуск

```bash
cd propis_app
python main.py
```

Откроется главное окно программы.

### 1.4 Обновление

Если вы обновили код из репозитория:

```bash
git pull
pip install -r propis_app/requirements.txt
```

## 2. Обзор интерфейса

### 2.1 Верхняя панель управления

Слева направо:

| Элемент | Назначение |
|---------|-----------|
| **Открыть PRN...** | Выбор файла данных со стенда |
| **Соль** | KDP или DKDP |
| **Кислота** | Нейтральный, 9.83%, 12.5%/12.67%, 18.5% |
| **Грань** | Призма {100} или Пирамида {101} |
| **Режим** | Классический, Современный или Оба (сравнение) |
| **Предобработка** | Галочка: включить/выключить конвейер предобработки |
| **Обработать** | Запустить обработку (активна после загрузки файла) |

### 2.2 Вкладки

Программа имеет 6 вкладок:

#### Вкладка «Сигнал»
Основная рабочая вкладка. Здесь вы:
- Видите сырой интерференционный сигнал от двух светодиодов
- Видите кривую температуры
- Настраиваете границы обработки (n1, n2, im, isat) перетаскиванием линий или вводом чисел

#### Вкладка «Предобработка»
Показывает сигнал до и после предобработки. Позволяет настроить параметры фильтров.

#### Вкладка «Кинетика»
Главный результат — кинетические кривые:
- R(ΔT) — скорость роста от переохлаждения (мм/день)
- R(σ) — скорость роста от перенасыщения (мкм/мин)
- Эталонные кривые для сравнения

#### Вкладка «Результаты»
Таблица выходных параметров (аналог Mat из Mathcad). Экспорт в CSV.

#### Вкладка «Сравнение»
Два графика рядом: классический и современный режим. Таблица различий параметров.

#### Вкладка «Пакетная»
Обработка всех PRN файлов в папке одним нажатием. Сводная таблица.

### 2.3 Строка состояния

Внизу окна — текущий статус: загруженный файл, количество отсчётов, диапазон температур, результаты обработки или сообщения об ошибках.

## 3. Пошаговая обработка файла

### Шаг 1. Загрузка данных

1. Нажмите **«Открыть PRN...»**
2. Выберите файл `.prn` из папки с данными эксперимента
3. В строке состояния появится информация: имя файла, количество отсчётов, диапазон температур
4. Вкладка «Сигнал» автоматически покажет загруженные данные

**Что вы видите на графике:**
- **Верхний график** — осциллирующий сигнал (синусоида с переменной амплитудой и частотой). Голубая линия — LED1 (470 нм), жёлтая — LED2 (590 нм)
- **Нижний график** — температура раствора (красная линия), монотонно растёт от ~45°C до ~49°C

### Шаг 2. Выбор типа соли и раствора

В верхней панели установите:

- **Соль**: KDP (дигидрофосфат калия) или DKDP (дейтерированный)
- **Кислота**: тип раствора по кислотности
- **Грань**: какая грань кристалла измерялась

Это влияет на:
- Коэффициенты растворимости C(T) для расчёта перенасыщения σ
- Коэффициенты пересчёта интерференционных полос в толщину (gran1, gran2)

### Шаг 3. Настройка границ обработки

Это **ключевой этап** — от правильности границ зависит точность результатов.

На вкладке «Сигнал» вы увидите 4 вертикальные пунктирные линии:

| Линия | Цвет | Что обозначает | Как настроить |
|-------|------|----------------|---------------|
| **n1** | зелёная | Начало рабочего участка | Поставьте после начальных артефактов (шум включения стенда) |
| **n2** | зелёная | Конец рабочего участка | Поставьте до конца записи или до момента выключения |
| **im** | красная | Левая граница зоны насыщения | Поставьте туда, где осцилляции начинают замедляться |
| **isat** | синяя | Правая граница зоны насыщения | Поставьте туда, где осцилляции полностью прекратились |

**Способы перемещения границ:**
- **Перетаскивание мышью**: ухватите пунктирную линию на графике и перетащите
- **Числовой ввод**: впишите номер отсчёта в соответствующее поле (n1, n2, im, isat)

**Как найти зону насыщения:**
1. Найдите участок, где осцилляции замедляются (период растёт) — кристалл перестаёт расти
2. Поставьте **im** чуть левее этого участка (где осцилляции ещё видны, но редкие)
3. Поставьте **isat** правее (где осцилляции полностью исчезли — начинается растворение)
4. Зона между im и isat — это переход от роста к растворению, где определяется tn

**Поле «Паразит»:** количество начальных полос, которые являются артефактами. Смотрите файл `прочитай.txt` в папке с данными — там оператор записал, сколько начальных полос паразитные (например, «1→6-паразит» означает первые 6 экстремумов — артефакты).

### Шаг 4. Выбор режима

- **Классический** — воспроизводит алгоритм из Mathcad (детекция экстремумов + степенной закон)
- **Современный** — использует преобразование Гильберта и BCF-модель
- **Оба (сравнение)** — запускает оба режима и показывает результаты рядом

Для первого знакомства рекомендуется **«Классический»** — его результаты можно сравнить с известными значениями из Mathcad.

### Шаг 5. Обработка

1. Нажмите **«Обработать»**
2. В строке состояния появится «Обработка...», затем результат:
   ```
   Готово: tn=48.60°C, Td=0.40°C, s1=-0.0068
   ```
3. Автоматически обновятся вкладки «Кинетика» и «Результаты»

### Шаг 6. Анализ результатов

Переключитесь на вкладку **«Кинетика»** — см. раздел 4 ниже.

### Шаг 7. Итерация

Если результат выглядит неправильно:
1. Вернитесь на вкладку «Сигнал»
2. Подвигайте границы im и isat
3. Нажмите «Обработать» снова
4. Повторяйте, пока кинетическая кривая не станет гладкой и физически осмысленной

## 4. Как читать результаты

### 4.1 График R(ΔT) — скорость роста от переохлаждения

**Оси:**
- **X**: переохлаждение ΔT = tn − T (°C). Чем правее — тем больше переохлаждение
- **Y**: скорость роста R (мм/день)

**Кривые:**
- **Точки (голубые/жёлтые)** — экспериментальные данные (канал 1 и канал 2)
- **Сплошная линия** — фиттинг (аппроксимация)
- **Зелёная пунктирная (Cfe=0)** — эталон чистого раствора
- **Красная пунктирная (Cfe=16ppm)** — эталон загрязнённого раствора

**Интерпретация:**
- Экспериментальная кривая **левее зелёной** → отличный раствор
- **Между зелёной и красной** → нормальный раствор
- **Правее красной** → плохой раствор, требует очистки от примесей

**Мёртвая зона** обозначена вертикальной пунктирной линией (фиолетовая): это минимальное переохлаждение, ниже которого кристалл не растёт.

### 4.2 График R(σ) — скорость роста от перенасыщения

**Оси:**
- **X**: перенасыщение σ (%)
- **Y**: скорость роста R (мкм/мин)

Этот график показывает ту же зависимость, но в единицах перенасыщения. Используется для фиттинга и определения параметров s1, s2.

### 4.3 Таблица параметров (вкладка «Результаты»)

| Параметр | Что означает | Типичные значения |
|----------|-------------|-------------------|
| **te** | Равновесная температура (°C) — максимальная T в рабочем участке | 48–50 |
| **tn** | Температура насыщения (°C) — при ней скорость роста = 0 | 48–49 |
| **Td** | Мёртвая зона (°C) — минимальное переохлаждение для начала роста | 0.1–1.0 |
| **s1** | Мёртвая зона (% перенасыщения) | −1 ... 0 |
| **s0** | Кинетический коэффициент — «скорость» кривой | 1–10 |
| **s2** | Параметр формы — наклон в log-log координатах | 0.2–0.5 |
| **Sig035** | σ при F > 0.35 (%) — характеристика мёртвой зоны | 1–3 |
| **w** | Показатель степени в формуле G(σ) = s₀·(σ − s₁)^w | обычно 1 |

### 4.4 Что означает «хороший» результат

Признаки правильной обработки:
- **Кинетическая кривая** — гладкая, монотонно растущая
- **Td** в пределах 0.1–1.0°C (не 3–5°C — это признак неверных границ)
- **Данные двух каналов** примерно совпадают (если сильно расходятся — проблема с сигналом)
- **Кривая между эталонами** — типичный раствор

Признаки проблемы:
- Разброс точек — сильный шум, попробуйте предобработку
- Кривая с изломами — неверные границы или паразитные полосы
- Td > 2°C — скорее всего неверно определена точка насыщения (подвигайте im/isat)

## 5. Предобработка сигнала

### 5.1 Когда включать

Предобработка **не нужна** для чистых сигналов (тестовые данные 020326–040326 обычно достаточно чистые).

Включайте предобработку, если:
- Сигнал сильно зашумлён (высокочастотные колебания поверх основной синусоиды)
- Базовая линия «плывёт» (медленный дрейф среднего значения)
- Амплитуда затухает со временем
- Есть скачки или разрывы в сигнале

### 5.2 Параметры предобработки

На вкладке **«Предобработка»** доступны следующие настройки:

| Параметр | По умолчанию | Назначение |
|----------|-------------|-----------|
| **Удаление тренда** | Вкл | Убирает медленный дрейф базовой линии |
| **Полосовой фильтр** | Вкл | Пропускает только частоты интерференционного сигнала |
| **f_low** | 0.005 Гц | Нижняя граница полосы (убирает дрейф) |
| **f_high** | 0.1 Гц | Верхняя граница полосы (убирает ВЧ шум) |
| **Нормализация** | Вкл | Делит сигнал на огибающую — выравнивает амплитуду |
| **Вейвлет** | Выкл | Дополнительное шумоподавление (для сильно зашумлённых данных) |

### 5.3 Как пользоваться

1. Загрузите файл
2. Перейдите на вкладку «Предобработка»
3. Настройте параметры
4. Нажмите **«Применить»**
5. Сравните синий (до) и жёлтый (после) сигналы
6. Если результат устраивает — поставьте галочку «Предобработка» в верхней панели и обработайте файл

### 5.4 Подбор частот фильтра

Частоты f_low и f_high зависят от скорости роста кристалла:
- **Быстрый рост** (частые осцилляции): f_high можно увеличить до 0.2 Гц
- **Медленный рост** (редкие осцилляции): f_low уменьшить до 0.001 Гц
- Если после фильтрации синусоида «обрезается» — расширьте полосу
- Если шум не убирается — сузьте полосу

## 6. Классический vs современный режим

### 6.1 Классический режим

Воспроизводит алгоритм Mathcad 2000:
- Находит максимумы и минимумы синусоидального сигнала
- Расстояние между экстремумами → скорость роста
- Фиттинг степенным законом: G(σ) = s₀·(σ − s₁)^w
- Даёт **дискретные** точки R (по одной на каждую пару экстремумов)

**Преимущества:** результаты напрямую сравнимы с Mathcad, проверенная методика.
**Недостатки:** ограниченное разрешение (одна точка на полосу), требует ручной настройки.

### 6.2 Современный режим

Использует аналитический сигнал (преобразование Гильберта):
- Мгновенная фаза φ(t) → мгновенная частота f(t) → скорость роста R(t)
- Фиттинг BCF-моделью: R = β·(σ − σ_d)² / (σ₁ + σ − σ_d)
- Даёт **непрерывную** кривую R(t) для каждого отсчёта

**Преимущества:** субполосное разрешение, автоматический, непрерывная кривая.
**Недостатки:** чувствителен к шуму (обязательна предобработка), результаты могут отличаться от Mathcad.

### 6.3 BCF-модель vs степенной закон

| Свойство | Степенной закон | BCF-модель |
|----------|----------------|------------|
| Формула | G = s₀·(σ − s₁)^w | R = β·(σ − σ_d)² / (σ₁ + σ − σ_d) |
| При малых σ | ∝ (σ − s₁)^w | ∝ (σ − σ_d)² (спиральный рост) |
| При больших σ | ∝ (σ − s₁)^w | ∝ (σ − σ_d) (шероховатый рост) |
| Параметры | s₀, s₁, w | β, σ_d, σ₁ |
| Физический смысл | Эмпирический | Теория Бартона—Кабреры—Франка |

BCF-модель лучше описывает переход от спирального к шероховатому росту при высоких перенасыщениях.

### 6.4 Режим «Оба (сравнение)»

Запускает оба режима и показывает:
- Два графика R(σ) рядом
- Таблицу различий параметров (tn, Td, s1 и т.д.)
- Помогает оценить, насколько современный метод отличается от классического

## 7. Пакетная обработка

Для обработки нескольких файлов одновременно:

1. Перейдите на вкладку **«Пакетная»**
2. Нажмите **«Выбрать папку...»** — выберите директорию с PRN файлами
3. Программа покажет количество найденных PRN файлов
4. Нажмите **«Обработать все»**
5. Прогресс-бар показывает текущий файл
6. По завершении — сводная таблица со всеми результатами

**Важно:** при пакетной обработке используются одинаковые границы (n1, n2, im, isat) для всех файлов. Это подходит для серии экспериментов с похожими условиями. Если условия сильно различаются — обрабатывайте файлы по отдельности.

**Цветовая кодировка:** файлы с ошибками обработки выделены красным.

## 8. Экспорт результатов

### 8.1 Экспорт параметров одного файла

На вкладке «Результаты»:
- **«Экспорт CSV»** — таблица параметров в формате CSV (открывается в Excel)
- **«Экспорт Mat»** — текстовый файл в формате, совместимом с записями Mathcad

### 8.2 Экспорт пакетной обработки

На вкладке «Пакетная»:
- **«Экспорт CSV»** — сводная таблица всех файлов в одном CSV

### 8.3 Формат CSV

```csv
Параметр,Значение,Описание
Salt,1,Тип соли (1=KDP, 2=DKDP)
te,48.85,Равновесная температура (°C)
tn,48.60,Температура насыщения (°C)
Td,0.40,Мёртвая зона (°C)
...
```

### 8.4 Формат Mat (текстовый)

```
# Propis IPFRAN — результаты обработки
# Файл: __030326_5.prn
Salt=1
Acid=0
te=48.85
tn=48.60
Td=0.40
s1=-0.0068
...
```

## 9. Справочник параметров

### 9.1 Входные параметры (задаёт оператор)

| Параметр | Диапазон | Описание | Как определить |
|----------|---------|----------|---------------|
| **n1** | 0–65000 | Начало рабочего участка | После начальных артефактов (обычно 50–200) |
| **n2** | n1–65000 | Конец рабочего участка | Перед концом записи или артефактами выключения |
| **im** | n1–n2 | Левая граница зоны насыщения | Где осцилляции замедляются (40–60% от n2) |
| **isat** | im–n2 | Правая граница зоны насыщения | Где осцилляции прекратились (50–70% от n2) |
| **Salt** | 1, 2 | Тип соли | 1 = KDP (KH₂PO₄), 2 = DKDP (KD₂PO₄) |
| **Acid** | 0–3 | Кислотность | 0 = нейтр., 1 = 9.83%, 2 = 12.5%, 3 = 18.5% |
| **Face** | 0, 1 | Грань кристалла | 0 = призма {100}, 1 = пирамида {101} |
| **Паразит** | 0–50 | Паразитные полосы | Из файла прочитай.txt |

### 9.2 Выходные параметры

| Параметр | Единицы | Описание | Физический смысл |
|----------|---------|---------|-----------------|
| **te** | °C | Равновесная температура | Макс. температура в рабочем участке |
| **tn** | °C | Температура насыщения | Температура при R = 0 (переход рост → растворение) |
| **Td** | °C | Мёртвая зона | tn − T_начала_роста. Чем больше — тем хуже раствор |
| **s1** | % | Мёртвая зона (перенасыщение) | σ при котором рост начинается. Обычно отрицательное |

## 10. Типичные проблемы и решения

### Проблема: Td получается слишком большой (> 2°C)

**Причина:** границы im/isat стоят неправильно — зона насыщения определена неточно.

**Решение:**
1. На вкладке «Сигнал» внимательно рассмотрите область, где осцилляции замедляются
2. Приблизьте этот участок (колёсико мыши для зума)
3. Поставьте im прямо перед тем, как последние осцилляции затихают
4. Поставьте isat сразу после прекращения осцилляций
5. Зона между im и isat должна быть относительно небольшой (500–2000 отсчётов)

### Проблема: слишком мало точек на кинетической кривой

**Причина:** алгоритм не находит достаточно экстремумов.

**Решение:**
- Расширьте рабочий участок (уменьшите n1, увеличьте n2)
- Уменьшите порог детекции пиков (если сигнал слабый)
- Попробуйте современный режим — он даёт непрерывную кривую

### Проблема: точки двух каналов сильно расходятся

**Причина:** один из каналов зашумлён или имеет артефакты.

**Решение:**
- Включите предобработку
- На вкладке «Кинетика» отключите зашумлённый канал (снимите галочку CH1 или CH2)
- Используйте данные только надёжного канала

### Проблема: кинетическая кривая с изломами/выбросами

**Причина:** паразитные полосы или скачки сигнала.

**Решение:**
- Проверьте файл `прочитай.txt` — сколько начальных полос паразитные
- Установите соответствующее значение в поле «Паразит»
- Включите предобработку (полосовой фильтр уберёт скачки)

### Проблема: программа не запускается (ошибка импорта PyQt6)

**Причина:** PyQt6 не установлен или нет дисплея.

**Решение:**
```bash
pip install PyQt6
```

Для WSL без графического дисплея: установите WSLg или используйте X-сервер (VcXsrv, Xming).

### Проблема: «No valid data rows found»

**Причина:** файл пустой, повреждён или в неожиданном формате.

**Решение:**
- Откройте PRN файл текстовым редактором
- Проверьте, что в нём 7 столбцов через пробелы
- Убедитесь, что кодировка файла — UTF-8 или ASCII

### Проблема: при пакетной обработке все файлы с ошибками

**Причина:** границы im/isat не подходят для всех файлов в папке.

**Решение:**
- Обработайте файлы по одному, подобрав границы для каждого
- Или выберите усреднённые границы, подходящие для большинства файлов
