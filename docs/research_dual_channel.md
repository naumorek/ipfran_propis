# Исследование: Двухканальная обработка сигналов интерферометра

**Дата:** 2026-03-14

---

## Физика системы

Луч разделяется **призмой** на два канала (LED1, LED2). Перед одним детектором стоит **поляризатор**. Соотношение ~40/60. Оба канала видят **одну и ту же фазу** φ(t):

```
LED1(t) = A1·cos(φ(t)) + DC1 + noise1(t)
LED2(t) = A2·cos(φ(t)) + DC2 + noise2(t)
```

Сейчас используется только `signal = LED1 / LED2` — деление убирает общий дрейф, но **теряет фазовую информацию каждого канала отдельно**.

---

## Что можно извлечь из двух каналов

### 1. Поточечная оценка шума (дифференциальный шум)

Если фазы одинаковы, то `φ1(t) - φ2(t)` = чистый шум:

```python
analytic1 = hilbert(led1)
analytic2 = hilbert(led2)
phase1 = unwrap(angle(analytic1))
phase2 = unwrap(angle(analytic2))
noise_phase = phase1 - phase2  # шум в каждой точке
```

**Применение:** вместо калибровки шума из dead zone — прямое измерение шума в каждый момент. Это решает проблему noise floor subtraction без приближений.

### 2. Усреднение фаз (SNR improvement)

```python
# Взвешенное по амплитуде усреднение
w1 = abs(analytic1)**2
w2 = abs(analytic2)**2
phase_avg = (w1 * phase1 + w2 * phase2) / (w1 + w2)
```

**Выигрыш:** +3 дБ SNR (√2 улучшение). Шумы каналов некоррелированы → усреднение подавляет.

### 3. Оптимальное Common-Mode Rejection (CMR)

Текущее LED1/LED2 — простейшее CMR. Можно лучше:

```python
# Оптимальный коэффициент подавления
alpha = cov(led1, led2) / var(led2)
signal_cmr = led1 - alpha * led2  # дифференциальный сигнал
```

Подавляет общемодовые помехи (дрейф питания, температура LED) на 20–30 дБ вместо ~15 дБ у простого деления.

### 4. Мониторинг когерентности (quality metric)

```python
freqs, coherence = scipy.signal.coherence(led1, led2, fs=1.0)
# coherence ≈ 1 → каналы синхронны (хороший сигнал)
# coherence < 0.5 → проблемы (сбой канала, шум доминирует)
```

По частотам:
- f < 0.01 Hz: когерентность ~0.9 (общий дрейф)
- 0.01–0.1 Hz: когерентность ~0.7 (зона полос интерференции)
- f > 1 Hz: когерентность ~0.3 (некоррелированный шум)

## Математические методы обработки

### A. Dual-Channel Hilbert (рекомендуется первым)

Hilbert отдельно на каждый канал → взвешенное усреднение фаз:

```python
def dual_hilbert_phase(led1, led2):
    a1 = hilbert(led1); a2 = hilbert(led2)
    p1 = unwrap(angle(a1)); p2 = unwrap(angle(a2))
    amp1 = abs(a1); amp2 = abs(a2)

    # Взвешенное усреднение (больше доверия каналу с большей амплитудой)
    w1 = amp1**2; w2 = amp2**2
    phase = (w1*p1 + w2*p2) / (w1 + w2 + 1e-10)

    # Дифференциальный шум (для диагностики)
    noise = abs(p1 - p2)

    return phase, (amp1 + amp2)/2, noise
```

**Плюсы:** простая реализация (scipy есть), +3 дБ SNR, поточечная оценка шума.
**Минусы:** Hilbert всё ещё шумит в dead zone при малой амплитуде обоих каналов.

### B. Квадратурная демодуляция (I/Q)

Если поляризатор создаёт ~90° сдвиг между каналами:

```python
def iq_demodulation(led1, led2):
    i = led1 - median(led1)  # In-phase
    q = led2 - median(led2)  # Quadrature
    phase = unwrap(arctan2(q, i))
    amplitude = sqrt(i**2 + q**2)
    return phase, amplitude
```

**Плюсы:** прямая фаза без Hilbert, нет проблемы dead zone (arctan2 определён при любых I,Q).
**Минусы:** требует точной калибровки угла между каналами. Если угол ≠ 90° — систематическая ошибка.

**Калибровка:** можно оценить автоматически из FFT основной гармоники обоих каналов.

### C. PLL (Phase-Locked Loop) на двух каналах

Программная PLL отслеживает фазу сигнала. В dead zone интегратор удерживает последнюю частоту → R плавно → 0.

```python
def dual_pll(led1, led2, f_center, bw=0.001):
    signal = (led1 + led2) / 2  # или CMR
    # ... стандартная PLL на усреднённом сигнале
```

**Плюсы:** отличная обработка dead zone (rated "Excellent" в research_signal_cleaning.md).
**Минусы:** требует настройки bandwidth и центральной частоты.

### D. CWT Ridge на двух каналах

Вейвлет-анализ каждого канала, выбор гребня с максимальной энергией в обоих:

```python
from ssqueezepy import cwt, extract_ridges, Wavelet
Wx1, scales = cwt(led1, Wavelet('gmw'))
Wx2, _ = cwt(led2, Wavelet('gmw'))
# Извлечь гребни, взвесить по энергии
```

**Плюсы:** автоматическое подавление dead zone, лучшее разрешение.
**Минусы:** новая зависимость (ssqueezepy), медленнее.

## Вопрос биморфизма (двулучепреломление KDP)

KDP — двупреломляющий кристалл: n_o ≈ 1.493, n_e ≈ 1.467, Δn ≈ 0.026.

**Можно ли измерить R_ordinary и R_extraordinary отдельно?**

Нет, по двум причинам:
1. **Малый фазовый сдвиг:** для d = 10 мкм → Δφ ≈ 0.13 рад ≈ 7.5° — неизмеримо мало
2. **Утечка поляризатора:** 10–30% кросс-поляризации размывает разделение

Кристалл растёт как целое — R_o и R_e определяются одним и тем же Δd/dt.

## Рекомендации по внедрению

### Этап 1: Немедленно (минимальные изменения)

1. **Заменить LED1/LED2 на оптимальное CMR** (~5 строк кода)
   - Ожидаемое улучшение: +5 дБ SNR
   - Риск: минимальный, обратно совместимо

2. **Добавить мониторинг когерентности** в отчёт
   - Флаг качества данных для каждого эксперимента

### Этап 2: Среднесрочно (dual-channel processing)

3. **Dual-Channel Hilbert с усреднением фаз**
   - +3 дБ SNR, поточечная оценка шума
   - Замена noise floor subtraction на дифференциальный шум

4. **Калибровка поляризаторов** (автоматическая из FFT)
   - Если угол ≈ 90° → квадратурная демодуляция (лучшая dead zone)
   - Если нет → weighted averaging

### Этап 3: Долгосрочно

5. **CWT Ridge на двух каналах** — лучшее разрешение + dead zone
6. **PLL** — если нужна максимальная стабильность в dead zone

### Что НЕ делать

- Не пытаться измерить R_ordinary vs R_extraordinary — невозможно
- Не применять I/Q без калибровки угла — систематическая ошибка
- Не игнорировать когерентность каналов — главный индикатор качества
